# \[Mobile] 링크를 메일프로그램으로 보내는 방법

### 개요 <a href="#a03" id="a03"></a>

이 매뉴얼 에서는 사용 중인 기기에 설치된 메일 소프트웨어를 통해 링크 정보를 보내는 방법에 대해 설명합니다.

***

### 제한·보충사항 <a href="#a04" id="a04"></a>

* iOS 애플리케이션과 Android 애플리케이션에서는 외부 메일을 여는 아이콘이 표시되지 않습니다.
* 사용자 로그인 페이지의 메인 도메인을 설정하는 방법에 따라 링크 URL의 도메인을 지정하면, 생성되는 링크 URL의 메인 도메인도 함께 변경됩니다.
* 사용자 로그인 페이지의 메인 도메인을 설정하는 방법으로 메인 도메인을 변경하더라도, 변경 이전에 생성된 Shared Box 링크에는 계속 접근할 수 있습니다.
*   링크에 대한 접근 횟수는 세션 단위로 집계되며, 그 횟수는 링크를 연 Web 브라우저에 Cookie로 저장됩니다. 따라서 한 사용자가 동일한 세션에서 링크에 10회 접근하더라도 접근 횟수는 1회로 계산됩니다.

    브라우저의 방문 기록을 삭제하거나, 브라우저를 연 시점으로부터 24시간이 지나면 해당 세션은 종료됩니다.
* 전송한 링크를 열 때, 두 번째 이후에도 비밀번호 입력을 요구하도록 설정할 수는 없습니다.
*   링크를 생성한 이후, 대상 폴더 또는 파일이 저장된 폴더에 대한 사용자 접근 권한이 제거되는 경우, 링크로 접속하더라도 해당 폴더나 파일에 접근할 수 없습니다.

    링크를 생성한 사용자의 접근 권한이 삭제되면, 링크를 받은 사용자 역시 해당 폴더나 파일에 접근할 수 없습니다.
*   링크 수신 화면에서 파일을 미리보기 할 때 표시되는 날짜와 시간은 파일의 업데이트 시각입니다. 이 날짜와 시간을 숨길 수는 없습니다.

    동일한 파일명으로 파일을 덮어쓰면, 링크 수신 화면에서도 해당 파일의 업데이트 시각이 변경됩니다.

***

### 절차 <a href="#a05" id="a05"></a>

1. [\[Mobile\] 링크 생성 화면을 표시하여 기본 기능을 확인하는 방법](https://help.directcloud.net/user_manual/file_share/m_link_check)에 따라, 메일로 전송하려는 폴더 또는 파일의 링크 화면을 엽니다.



2. 링크 화면 왼쪽 아래에 있는 다른 방법으로 열기 아이콘을 탭합니다.\
   사용할 수 있는 애플리케이션 목록이 표시됩니다.



3. 링크 정보를 추가하고 싶은 메일 애플리케이션을 탭합니다.



4. 1회만 또는 항상 중에서 선택하여 탭합니다. 메일 작성 화면에 링크 정보가 자동으로 추가됩니다.
