# \[Mobile] 링크를 수신할 때, 이메일 주소 입력을 통해 본인 확인을 진행하도록 설정하는 방법

### 개요 <a href="#a03" id="a03"></a>

이 매뉴얼에서는 iOS 애플리케이션 또는 Android 애플리케이션의 링크 화면에서, 링크를 받은 사용자가 링크 수신 화면에 접근할 때 이메일 주소 입력을 통해 본인 확인을 수행하도록 설정하는 방법에 대해 설명합니다.

***

### 제한·보충사항 <a href="#a04" id="a04"></a>

* 이 매뉴얼의 설정은 My Box의 파일을 선택하여 생성한 링크에도 적용됩니다.
* 링크 수신 화면에 접근할 수 있는 것은 링크 화면의 수신자 항목에 입력한 이메일 주소와 일치하는 경우에만 가능합니다.
* 이 매뉴얼의 절차에 따라 메일을 전송하면, 이후 링크 화면에서 이력 중 링크를 선택할 때 링크 복사, URL 변경 아이콘이 표시되지 않아 링크 설정을 변경할 수 없게 됩니다.
* 링크 수신 화면에 접근할 때 2단계 인증을 사용하여 본인 확인을 수행하도록 설정할 수는 없습니다.
* 링크 화면에 표시되는 언어는 프로필의 설정 화면에서 변경할 수 있습니다.
* 링크를 생성한 이후 대상 폴더 또는 파일이 저장된 폴더에 대한 사용자 접근 권한이 삭제되면 링크로 접속하더라도 해당 폴더나 파일에 접근할 수 없습니다.\
  또한 링크를 생성한 사용자의 접근 권한이 삭제되면, 링크 수신자도 해당 폴더나 파일에 접근할 수 없게 됩니다.
* [\[Mobile\] 링크 생성 화면을 표시하여 기본 기능을 확인하는 방법](https://help.directcloud.net/user_manual/file_share/m_link_check)의 상세 설정에서 비밀번호가 설정된 경우, 먼저 비밀번호 입력 화면이 표시된 이후 수신자 확인 화면이 나타납니다.
*   링크 접근 횟수는 세션 단위로 집계되며, 해당 횟수는 링크를 연 Web 브라우저의 Cookie로 저장됩니다.\
    따라서 동일한 사용자가 같은 세션에서 링크를 10회 열어도 접근 횟수는 1회로 계산됩니다.

    브라우저의 방문 기록을 삭제하거나, 브라우저를 연 시점으로부터 24시간이 지나면 해당 세션은 종료됩니다.
* 전송한 링크를 열 때 두 번째 이후에도 비밀번호 입력을 요구하도록 설정할 수는 없습니다.

***

### 절차 <a href="#a05" id="a05"></a>

{% hint style="warning" %}
**iOS 애플리케이션에서 설정하기**
{% endhint %}

1. [\[Mobile\] 링크 생성 화면을 표시하여 기본 기능을 확인하는 방법](https://help.directcloud.net/user_manual/file_share/m_link_check)의 순서로 공유 링크의 수신 화면에 액세스 할 때 이메일 주소를 입력하도록 설정하려는 폴더 또는 파일의 '링크' 화면을 표시합니다.



2. 링크 화면을 아래로 스크롤한 다음 메일 옵션을 탭합니다.\
   메일 옵션 화면이 표시됩니다.



3. '수신자 확인' 설정의 '링크 접속 시 수신자 메일주소 입력'의 토글 스위치를 켜고 오른쪽 상단의 '완료' 버튼을 누릅니다.



{% hint style="warning" %}
**Android 애플리케이션에서 설정**
{% endhint %}

1. [\[Mobile\] 링크 생성 화면을 표시하여 기본 기능을 확인하는 방법](https://help.directcloud.net/user_manual/file_share/m_link_check)의 순서로 공유 링크의 수신 화면에 액세스 할 때 이메일 주소를 입력하도록 설정하려는 폴더 또는 파일의 "링크" 화면을 표시합니다.



2. 링크 화면을 아래로 스크롤한 다음 메일 옵션을 탭합니다.\
   메일 옵션 화면이 표시됩니다.



3. '수신자 확인' 설정의 '링크 접속 시 수신자 메일주소 입력'의 토글 스위치를 켜고 오른쪽 상단의 '완료' 버튼을 누릅니다.
