# 사용자 페이지의 특정 폴더에서 업로드 알림을 활성화 하는 방법

### 개요 <a href="#a03" id="a03"></a>

이 매뉴얼에서는 사용자 페이지의 특정 폴더에서 업로드 알림을 사용하는 방법에 대해 설명합니다.

***

### 제한·보충사항 <a href="#a04" id="a04"></a>

* 폴더 속성을 설정하는 사용자는 <마스터> 접근 권한이 부여되어 있어야 합니다.\
  자세한 내용은 [접근 권한의 종류와 Web 브라우저에서 가능한 조작](https://help.directcloud.net/user_manual/access_permission/w_permission)을 참조해주시기 바랍니다.
* 메일 알림 기능은 첫 번째 계층의 폴더에 설정하며, 설정한 내용은 하위 폴더에도 그대로 적용됩니다.
* DirectCloud-CONNECT를 이용할 수 있는 경우, 사용자 페이지에 Connect 메뉴가 표시됩니다.
* DirectCloud-SHIELD DLP를 사용 중인 경우, 사용자 페이지에 DLP 메뉴가 표시됩니다.
* Warm Storage를 사용 중인 경우, 사용자 페이지에 Warm Storage 메뉴가 표시됩니다.\
  또한 Warm Storage에서는 업로드 시 메일 알림이 전송되지 않으므로, Warm Storage 폴더의 폴더 속성에는 메일 알림 항목이 표시되지 않습니다.
* 메일 알림 관련 폴더 속성은 Connect 폴더, DLP 폴더에서도 설정할 수 있으며, 해당 기능의 사양과 제약사항은 Shared Box와 동일합니다.

***

### 절차 <a href="#a05" id="a05"></a>

1. Shared Box, Connect, DLP 메뉴 중 하나를 클릭한 뒤, 메일 알림을 설정할 첫 번째 계층의 폴더를 선택합니다.

<figure><img src="../../.gitbook/assets/image (205).png" alt=""><figcaption></figcaption></figure>



2. 슬라이드 화면의 폴더 속성 설정에서 '이메일 통지' 항목이 사용함으로 표시되어 있는지 확인합니다.\
   사용하지 않음으로 표시되어 있는 경우, 오른쪽에 표시된 설정 버튼을 클릭합니다.

<figure><img src="../../.gitbook/assets/image (207).png" alt=""><figcaption></figcaption></figure>



3. 메일 알림 설정 화면이 표시됩니다.\
   사용함을 선택하고, 알림 메일을 보낼 간격을 선택한 뒤 확인 버튼을 클릭합니다.
