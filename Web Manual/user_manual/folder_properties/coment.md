# 사용자 페이지의 특정 폴더에서 파일 코멘트 기능을 활성화 하는 방법

### 개요 <a href="#a03" id="a03"></a>

이 매뉴얼에서는 사용자 페이지의 특정 폴더에서 파일 코멘트 기능을 활성화하는 방법에 대해 설명합니다.

***

### 제한·보충사항 <a href="#a04" id="a04"></a>

* 폴더 속성을 설정하는 사용자는 <마스터> 접근 권한이 부여되어 있어야 합니다.\
  자세한 내용은 [접근 권한의 종류와 Web 브라우저에서 가능한 조작](https://help.directcloud.net/user_manual/access_permission/w_permission)을 참조해주시기 바랍니다.
* 파일 코멘트는 첫 번째 계층과 두 번째 계층의 폴더에 설정하며, 설정된 내용은 하위 폴더에도 그대로 적용됩니다.
* 파일 코멘트 관련 폴더 속성은 Connect 폴더, DLP 폴더, 문서 관리 폴더에서도 설정할 수 있으며, 해당 기능의 사양과 제약 사항은 Shared Box와 동일합니다.
* 비즈니스 이상 요금제를 사용하면서 DirectCloud의 유료 옵션인 DirectCloud-SHIELD DLP를 이용 중인 경우, 사용자 페이지에 DLP 메뉴가 표시됩니다.
* 관리자가 DirectCloud AI를 사용으로 설정한 경우, 폴더 속성에 DirectCloud AI 항목이 표시되며, 동시에 버전 관리와 DirectCloud-OCR 항목은 폴더 속성에서 사라집니다.
* DirectCloud-CONNECT를 이용할 수 있는 경우, 사용자 페이지에 Connect 메뉴가 표시됩니다.\
  단, Connect 폴더의 폴더 속성에서는 워크플로우, SHIELD, 링크 설정을 사용할 수 없습니다.
* Warm Storage를 이용 중인 경우, 사용자 페이지에 Warm Storage 메뉴가 표시됩니다.\
  Warm Storage 폴더의 폴더 속성에는 코멘트 항목이 표시되지 않습니다.
* 관리자가 파일 코멘트를 사용으로 설정한 경우라도 Warm Storage의 파일에는 새로운 코멘트를 추가할 수 없습니다.\
  다만, 이미 등록된 파일 코멘트는 확인할 수 있습니다.

***

### 절차 <a href="#a05" id="a05"></a>

1. "Shared Box"  메뉴를 클릭하고 파일 주석을 설정하려는 첫 번째 또는 두 번째 계층 폴더를 클릭합니다.

<figure><img src="../../.gitbook/assets/image (196).png" alt=""><figcaption></figcaption></figure>



2. 슬라이드 화면의 폴더 속성 설정에서 '코멘트' 에 '사용함'이 표시되는지 확인합니다.\
   '사용 안함'이 표시되면 오른쪽에 표시된 설정 버튼을 클릭합니다.\
   코멘트 설정이 표시됩니다.

<figure><img src="../../.gitbook/assets/image (197).png" alt=""><figcaption></figcaption></figure>



3. '사용함'을 선택하고 확인 버튼을 클릭합니다.
