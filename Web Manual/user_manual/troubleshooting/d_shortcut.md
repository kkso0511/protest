# DirectCloud 드라이브의 바로가기 기능 관련 문제 해결 방법

### 개요 <a href="#a03" id="a03"></a>

DirectCloud 드라이브에서는 Shared Box에 저장된 폴더나 파일의 바로가기를 생성할 수 있습니다.\
이 매뉴얼에서는, DirectCloud 드라이브의 바로가기와 관련된 문제를 해결하는 방법에 대해 설명합니다.

***

### 절차 <a href="#a05" id="a05"></a>

{% hint style="warning" %}
**생성한 바로가기에서 다른 파일이 열리는 경우**
{% endhint %}

DirectCloud 드라이브 상의 폴더 또는 파일에 대해 바로가기를 생성한 후 로컬 PC로 이동한 경우, 바로가기를 통해 폴더나 파일에 접근하려고 하면 다른 폴더나 파일이 열리는 일이 발생할 수 있습니다.\
이 문제는 바로가기의 링크 대상이 자동으로 변경되는 것이 원인입니다.\
DirectCloud 드라이브에 로그인한 상태에서는 올바른 폴더 또는 파일 경로에 접근할 수 있지만, DirectCloud 드라이브에서 로그아웃한 상태로 바로가기에 접근하면 DirectCloud 드라이브가 마운트되어 있지 않기 때문에 접근할 수 없습니다.\
또한 Windows의 바로가기 추적 기능으로 인해 로컬 PC 상에서 동일한 이름의 폴더나 파일이 검색되고, 발견된 경우 로컬로 이동된 것으로 인식되어 바로가기의 링크 대상이 자동으로 변경됩니다.\
대처 방법으로는 아래의 두 가지가 있습니다.

* DirectCloud 드라이브 상의 폴더 및 파일에 대해 바로가기를 생성하지 않도록 합니다.
* Windows 설정에서 바로가기 추적 기능을 끕니다.\
  다만 이 방법은 로컬 PC의 레지스트리 값을 변경해야 하며 위험을 수반하므로, DirectCloud에서는 권장하지 않습니다.



{% hint style="warning" %}
**생성한 바로가기에 접근했을 때 오류가 표시되는 경우**
{% endhint %}

DirectCloud 드라이브 상의 파일에 대해 바로가기를 생성한 후 해당 바로가기를 PC로 이동한 경우, 바로가기를 통해 파일을 열려고 하면 파일에 접근할 수 없다는 내용의 오류가 표시될 수 있습니다.\
이 원인으로는 일시적으로 네트워크가 차단되었을 가능성이 있습니다.\
네트워크 차단으로 인해 Windows가 DirectCloud 드라이브의 파일 경로를 인식하지 못하게 되면 위와 같은 오류가 발생합니다.\
이 경우 네트워크가 복구된 후 DirectCloud 드라이브에 다시 로그인하면 바로가기를 통해 다시 접근할 수 있게 됩니다.



{% hint style="warning" %}
**DirectCloud 드라이브 바로 가기 아이콘에 물음표가 표시되는 경우**
{% endhint %}

PC에서 DirectCloud 드라이브의 바로가기를 생성한 후, 한 번 DirectCloud 드라이브에서 로그아웃한 다음 DirectCloud 드라이브 바로가기의 이름을 변경하면, DirectCloud 드라이브의 바로가기에 물음표(물음표 표시)가 표시되게 됩니다.\
이 상태에서도 바로가기를 통해 DirectCloud 드라이브에 접근하는 것은 가능합니다.\
바로가기의 이름을 원래대로 되돌리면 물음표는 사라집니다.
