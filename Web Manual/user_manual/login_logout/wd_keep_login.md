# \[Web, Drive] 저장된 로그인 정보로 DirectCloud를 사용하는 방법

### 개요 <a href="#a03" id="a03"></a>

DirectCloud의 애플리케이션에서는 로그인할 때 입력한 회사코드, 사용자ID, 비밀번호를 저장할 수 있습니다. 따라서 로그아웃 하거나 액세스 토큰의 유효기간이 만료되지 않는 한, 회사코드, 사용자ID, 비밀번호의 입력을 생략할 수 있습니다.\
이 매뉴얼에서는 로그인할 때 회사코드, 사용자ID, 비밀번호를 저장하는 방법에 대해 설명합니다.

***

### 제한·보충사항 <a href="#a04" id="a04"></a>

* 이 매뉴얼에서는 'Web 브라우저'와 'DirectCloud 드라이브 (Windows)'의 작동 절차에 대해 설명하지만 다른 응용 프로그램도 같은 방법으로 설정할 수 있습니다.
* 관리자 페이지의 '보안정책' > '로그인 정책' 메뉴에서 '로그인 자동종료 설정' 을 '사용'으로 설정하면 액세스 토큰의 '60일' 만료일 보다 일찍 로그아웃 되도록 설정 할 수 있습니다.
*   DirectCloud 드라이브의 경우 액세스 토큰 만료일로부터 10일 이내에 작업을 수행하면 액세스 토큰이 갱신되지만 DirectCloud 드라이브가 아닌 PC Agent의 경우 작업을 수행할 때 액세스 토큰이 갱신되지 않습니다.

    따라서 로그인 정보를 유지하도록 이 문서의 단계를 설정하여 DirectCloud 드라이브 이외의 애플리케이션에 로그인하면 액세스 토큰이 만료되면 자동으로 로그아웃 됩니다.
*   사용자 페이지에서 로그아웃 하여 DirectCloud 드라이브를 '종료'한 경우 애플리케이션을 다시 시작할 때 저장된 로그인 정보와 함께 DirectCloud 드라이브를 사용할 수 있습니다.

    로그아웃 하면 로그인 정보가 삭제되므로 DirectCloud 드라이브를 다시 시작하면 로그인 화면이 나타납니다.
* DirectCloud 드라이브를 사용하는 경우 이 문서의 단계에서 '로그인 유지'를 선택하여 로그인하고 네트워크에 연결할 수 없으면 오프라인으로 전환됩니다. 오프라인 로그인 상태는 로그아웃 할 때까지 유지됩니다.
*   관리 페이지에서 2단계 인증을 활성화한 경우 일회용 비밀번호를 저장하여 일정 기간 동안 일회용 비밀번호로 인증을 건너뛸 수 있습니다. 단, '2단계 인증 유효 기간'을 설정하고 있는 경우, 일회용 비밀번호 유효 기간이 지나면 강제로 로그아웃 하게 됩니다.

    이 문서의 단계에서 로그인을 건너뛰면 예외 없이 로그아웃 되지만 2단계 인증을 다시 사용하여 '회사코드', '사용자 ID' 또는 '비밀번호'를 입력하지 않고 로그인할 수 있습니다.

***

### 절차 <a href="#a05" id="a05"></a>

{% hint style="warning" %}
**Web 브라우저**
{% endhint %}

1. 로그인 화면에서 회사코드, 사용자 ID, 비밀번호를 입력하고 로그인 상태 유지를 선택하고 로그인 버튼을 클릭합니다.<br>

<figure><img src="../../.gitbook/assets/image (1244).png" alt=""><figcaption></figcaption></figure>

로그인에 성공하면 입력한 회사코드, 사용자 ID, 비밀번호가 기억됩니다. 응용 프로그램에서 로그아웃 할 때까지 로그인 정보가 유지되므로 이후에 응용 프로그램에 액세스해도 로그인 화면이 표시되지 않습니다.



{% hint style="warning" %}
**DirectCloud 드라이브(Windows)**
{% endhint %}

1. 로그인 화면에서 회사코드, 사용자 ID, 비밀번호를 입력하고 로그인 상태 유지를 선택하고 로그인 버튼을 클릭합니다.

<figure><img src="../../.gitbook/assets/image (1245).png" alt=""><figcaption></figcaption></figure>

로그인에 성공하면 입력한 회사코드, 사용자 ID, 비밀번호가 기억됩니다.\
응용 프로그램에서 로그아웃 할 때까지 로그인 정보가 유지되므로 이후에 응용 프로그램을 시작하면 DirectCloud 드라이브가 로그인 상태가 됩니다.
