# 알림 조건을 충족하고 있음에도 알림 메일이 도착하지 않는 경우의 대처 방법

### 개요 <a href="#a03" id="a03"></a>

DirectCloud에서는 Amazon SES를 통해 알림 메일이 전송되지만, 수신 측 메일 서버의 설정으로 인해 메일을 수신하지 못하는 경우가 있습니다.\
본 매뉴얼에서는, 사용자 페이지에서 알림 메일을 수신할지 여부를 설정할 수 있도록 하는 방법 의 알림 조건을 충족했음에도 알림 메일이 도착하지 않는 원인과, 그에 대한 대처 방법에 대해 설명합니다.

***

### 제한·보충사항 <a href="#a04" id="a04"></a>

* 관리자 권한이 부여된 사용자는 관리자 페이지의 '계정 관리'를 사용할 수 있습니다.

***

### 절차 <a href="#a05" id="a05"></a>

{% hint style="warning" %}
**수신 메일 서버에 필요한 도메인이 허용되지 않음**
{% endhint %}

Amazon SES는 이중화 구성으로 되어 있어, 여러 개의 IP 주소가 사용되고 있습니다.\
사용 중인 메일 서버에서, 아래 도메인으로부터의 수신을 허용해 주시기 바랍니다.\
\
\*.us-west-2.amazonses.com



{% hint style="warning" %}
**Amazon SES 필터에 등록됨**
{% endhint %}

다음으로 인해 Amazon SES에 대한 반송 메일이 많아 필터에 등록되었을 수 있습니다.

* 사용자 정보로 입력한 이메일 주소에 오류가 있는 경우
* 받는 메일 서버에 일시적인 오류가 발생하여 수신할 수 없는 경우
* 수신자 메일 서버의 여유 공간이 부족하여 수신할 수 없는 경우
* 수신측의 메일 서버가 수신할 수 있는 용량을 초과한 파일이 첨부되어 있기 때문에 수신할 수 없는 경우
* 수신 메일 서버에서 'no-reply@directcloud.net'에서 보낸 메일을 차단하고 있는 경우
* 이메일 자동 응답이 설정된 경우
* 'no-reply@directcloud.net'으로 메일을 발송함

이 경우 관리자 페이지의 '계정 관리' -> '사용자' 항목에 들어가, 메일 수신이 되지 않고 있는 사용자 계정의 편집 화면에 들어가 '이메일 비활성화' 처리가 되어 있는지 확인합니다.\
이메일 비활성화가 되어있다면, 인증을 통해 해결합니다.



{% hint style="warning" %}
**사용자의 권한이 <업로더>인 경우**
{% endhint %}

사용자에게 <업로더> 권한이 부여된 경우, 알림 이메일이 전송되지 않습니다.&#x20;
