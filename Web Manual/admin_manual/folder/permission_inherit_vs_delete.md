# 접근 권한을 상속하지 않는 경우와, 접근 권한을 삭제한 경우의 차이

### 개요 <a href="#a03" id="a03"></a>

하위 폴더에서 상위 폴더의 접근 권한을 자동으로 반영하지 않고 새로운 접근 권한을 설정하는 방법에는 추가적으로 사용자 접근 권한 설정 화면 또는 Guest 접근 권한 설정 화면에서 사용자·그룹·Guest의 접근 권한을 삭제하는 방법이 있습니다.\
다만 해당 방법을 사용할 경우, 접근 권한을 삭제한 뒤에 상위 폴더의 접근 권한을 상속하지 않음의 체크를 해제하더라도, 상위 폴더의 접근 권한은 상속되지 않습니다.\
이 매뉴얼에서는 상위 폴더의 접근 권한을 상속하지 않는 폴더와, 사용자의 접근 권한을 삭제한 폴더의 차이점에 대해 설명합니다.

***

### 제한·보충사항 <a href="#a04" id="a04"></a>

* 관리자 권한이 부여된 사용자는 관리자 페이지에서 '공유 설정' 메뉴를 사용할 수 있습니다.
* 사용자 접근 권한 설정 화면에서 지정한 상위 단계 접근 권한 상속 설정은 Guest에도 적용됩니다.
* Guest 접근 권한 설정 화면에서는 상위 단계 접근 권한 상속을 설정할 수 없습니다.
* 접근 권한 삭제 등으로 인해 사용자에게 내부적으로 부여된 접근 권한 레벨(noAccess)은 폴더의 접근 권한 정보를 일괄로 내보내는 방법을 통해 확인할 수 있습니다.

***

### 절차 <a href="#a05" id="a05"></a>

{% hint style="warning" %}
**권한 설정 화면 표시**
{% endhint %}

1. '공유 설정' > 'Shared Box' 관리 메뉴를 선택하고 권한을 상속하지 않으려는 폴더를 클릭합니다.

<figure><img src="../../.gitbook/assets/image (2299).png" alt="" width="398"><figcaption></figcaption></figure>



2. 구성원 관리에서 그룹/사용자의 오른쪽에 표시되는 권한관리 버튼을 클릭합니다.\
   &#x20;권한 설정 화면이 표시됩니다.

<figure><img src="../../.gitbook/assets/image (2300).png" alt=""><figcaption></figcaption></figure>

&#x20;  &#x20;

{% hint style="warning" %}
**접근 권한을 상속하는 폴더의 경우**
{% endhint %}

상위 폴더의 접근 권한을 상속하는 폴더의 경우, 권한 설정 배경이 파란색으로 표시됩니다.\
또한 "상위계층 권한을 승계중입니다."라고 표시됩니다.\
이때의 상태는, 이하와 같습니다.

* 상위 폴더의 권한이 자동으로 반영됩니다.
* 선택한 폴더 자체에는 별도권한이 설정되지 않은 상태입니다.
* 상위 폴더의 권한과는 별도로 자신의 접근 권한을 추가할 수도 있습니다.

<figure><img src="../../.gitbook/assets/image (2301).png" alt=""><figcaption></figcaption></figure>



{% hint style="warning" %}
**권한을 상속하지 않는 폴더의 경우**
{% endhint %}

이 매뉴얼에서는 상위 폴더의 접근 권한을 상속하지 않도록 설정했을 때, 상위 폴더와 상속하도록 설정된 하위 폴더가 어떤 접근 권한 상태가 되는지에 대해 설명합니다.

**\[설정 폴더]**

사용자 접근 권한 설정 화면에서 '상위계층 권한 승계 안함'에 체크되어 있는 경우에는 상위 폴더의 접근 권한이 상속되지 않습니다. \
오른쪽 영역에서 접근 권한 설정 표시가 사라지고, '상위계층 권한을 승계하지 않습니다.' 라고 표시됩니다.\
이때의 접근 권한 상태는 다음과 같습니다:

* 상위 폴더의 접근 권한은 반영되지 않습니다.
* 선택한 폴더 자체에는 접근 권한이 설정되지 않은 상태입니다.
* 해당 폴더에 새로운 접근 권한을 설정할 수 있습니다.

<figure><img src="../../.gitbook/assets/image (2302).png" alt=""><figcaption></figcaption></figure>

\
**\[하위 폴더]**

상위 단계 폴더의 접근 권한을 상속하지 않도록 설정하더라도, 그보다 더 하위에 있는 폴더에는 '상위계층 권한을 승계중입니다.' 라고 표시됩니다.\
이때의 접근 권한 상태는 다음과 같습니다:

* 상위에서 설정한 폴더의 접근 권한이 하위 폴더에 반영됩니다.
* 하위 폴더 자체에는 접근 권한이 설정되지 않은 상태입니다.
* 해당 하위 폴더에서도 상위 단계의 접근 권한을 상속하지 않도록 다시 설정하거나, 새로운 접근 권한을 설정할 수 있습니다.

<figure><img src="../../.gitbook/assets/image (2303).png" alt=""><figcaption></figcaption></figure>



{% hint style="warning" %}
**사용자의 접근 권한을 삭제한 폴더의 경우**
{% endhint %}

사용자의 접근 권한을 삭제했을 때, 설정한 폴더와 그 하위 폴더가 어떤 접근 권한 상태가 되는지에 대해 설명합니다.

<figure><img src="../../.gitbook/assets/image (2304).png" alt=""><figcaption></figcaption></figure>

<h4 align="center">⬇️</h4>

<figure><img src="../../.gitbook/assets/image (2534).png" alt=""><figcaption></figcaption></figure>



**\[설정 폴더]**

접근 권한이 삭제된 사용자에게는 내부적으로 noAccess(접근 권한이 없는 사용자)라는 접근 권한 레벨이 부여됩니다.\
한 번 접근 권한이 삭제된 사용자에 대해서는 상위 단계의 접근 권한을 상속하지 않음에 체크한 뒤 다시 체크를 해제하더라도 상위 단계 폴더의 접근 권한이 상속되지 않습니다.

<figure><img src="../../.gitbook/assets/image (2306).png" alt=""><figcaption></figcaption></figure>

### &#x20;                                                         **⬇️**

<figure><img src="../../.gitbook/assets/image (2307).png" alt=""><figcaption></figcaption></figure>



**\[하위 폴더]**

상위 폴더에서 사용자의 접근 권한을 삭제한 경우, 더 하위에 있는 폴더의 접근 권한 상태는 다음과 같습니다:

* 상위에서 설정한 폴더의 접근 권한이 하위 폴더에 반영되며, 사용자의 접근 권한은 삭제된 상태로 유지됩니다.
* 하위 폴더에서도 상위 폴더의 접근 권한을 상속하지 않도록 설정하거나, 해당 폴더에 새로운 접근 권한을 설정할 수 있습니다.
* 상위 폴더의 접근 권한을 상속하지 않음 체크를 해제하더라도 삭제된 사용자의 접근 권한은 다시 표시되지 않습니다.
