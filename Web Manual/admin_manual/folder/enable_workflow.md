# 특정 폴더에서 워크플로우를 활성화하는 방법

### 개요 <a href="#a03" id="a03"></a>

이 매뉴얼에서는 특정폴더를 선택하여 워크플로우를 사용하도록 설정하는 방법에 대해 설명합니다.

***

### 제한·보충사항 <a href="#a04" id="a04"></a>

* 이 매뉴얼에서는 Shared Box 메뉴의 설정 방법을 설명하고 있지만,  DLP 관리 메뉴에서도 동일한 절차로 작업할 수 있습니다.
* 관리자 권한이 부여된 사용자는 관리자 페이지의 '공유 설정'을 사용할 수 있습니다.
* 초기 상태에서는 워크플로우가 '사용안함'으로 설정되어 있습니다.
* 워크플로우는 1단계 폴더에 설정하며, 설정 내용은 하위 폴더에 상속됩니다.
* [워크플로우를 활성화하는 방법](https://help.directcloud.net/admin_manual/workflow/enable_workflow)에서 워크플로우를 활성화하면 폴더 속성에 '워크플로우'가 표시됩니다.
* 워크플로우는 web 애플리케이션에서 생성해야 합니다.
* 애플리케이션별로 지원되는 기능이 다르기 때문에 지원되지 않는 기능에 대한 승인은 신청할 수 없습니다. 자세한 내용은 아래 문서를 참고하시기 바랍니다.
  * [다운로드 워크플로우를 신청하는 방법](https://help.directcloud.net/user_manual/workflow/download_workflow_apply)
  * [링크 전송 워크플로우를 신청하는 방법](https://help.directcloud.net/user_manual/workflow/link_workflow_apply)
  * [파일첨부 메일 발송 워크플로우를 신청하는 방법](https://help.directcloud.net/user_manual/workflow/mail_workflow_apply)
  * [업로드 워크플로우를 신청하는 방법](https://help.directcloud.net/user_manual/workflow/upload_workflow_apply)
* 워크플로우 신청 승인 기능은 Mac 버전 DirectCloud 드라이브를 제외한 모든 애플리케이션에서 이용할 수 있습니다.
* [워크플로우를 활성화하는 방법](https://help.directcloud.net/admin_manual/workflow/enable_workflow)에서 워크플로우를 활성화했더라도, 본 매뉴얼의 절차에서 워크플로우가 '사용 안함'으로 설정되어 있는 경우에는 DirectCloud 드라이브에서 다음 작업이 제한되지 않습니다:
  * 드래그 앤 드롭을 통한 다운로드 및 업로드
  * 직접 편집
  * 이름 변경
  * 폴더 및 파일 이동
  * 폴더 및 파일 복사
  * 폴더 생성
  * 폴더 및 파일 삭제
* [워크플로우를 활성화하는 방법](https://help.directcloud.net/admin_manual/workflow/enable_workflow)에서 워크플로우가 활성화되어 있는 경우, 본 매뉴얼의 절차에서 워크플로우를 사용하지 않음으로 설정해도 모바일 애플리케이션에서는 폴더에 다음 제한이 적용됩니다:
  * 관리자 페이지의 '보안 정책' > '모바일 보안' 메뉴에서 이메일 전송, 다른 앱으로 열기 기능이 사용하지 않음으로 고정됨
* DirectCloud-CONNECT를 사용할 수 있는 경우, 관리자 페이지에 Connect 관리 메뉴가 표시됩니다.
* DirectCloud-SHIELD DLP를 계약하고 있는 경우, 관리자 페이지에 DLP 관리 메뉴가 표시됩니다.
* Warm Storage를 사용할 수 있다면 관리자 페이지에 Warm Storage 관리 메뉴가 표시됩니다.
* Cold Storage를 계약하고 있는 경우, 관리자 페이지에 Cold Storage 관리 메뉴가 표시됩니다.
* Connect, Warm Storage, Cold Storage에서는 워크플로우를 사용할 수 없기 때문에 Connect 관리, Warm Storage 관리, Cold Storage 관리 메뉴에서 폴더를 선택해도 폴더 속성에 워크플로우가 표시되지 않습니다.

***

### 절차 <a href="#a05" id="a05"></a>

1. '공유 설정' > 'Shared Box' 관리 메뉴를 선택하고 워크플로우를 활성화할 폴더를 클릭합니다.

<figure><img src="../../.gitbook/assets/image (1956).png" alt=""><figcaption></figcaption></figure>



2. 화면을 아래로 스크롤하고 폴더 속성 설정에서 '워크플로우'의 '수정' 버튼을 클릭합니다.

<figure><img src="../../.gitbook/assets/image (1958).png" alt=""><figcaption></figcaption></figure>



3. 워크플로우 설정의 사용여부에서 '사용'을 선택하고 저장 버튼을 클릭합니다.

<figure><img src="../../.gitbook/assets/image (1959).png" alt=""><figcaption></figcaption></figure>
