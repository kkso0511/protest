# 관리자 페이지에 로그인할 수 없는 경우 해결 방법

### 개요 <a href="#a03" id="a03"></a>

관리자 페이지에 로그인할 수 없는 경우에는 발생 상황을 확인하고, 최적의 대응 방법을 시도해 보시기 바랍니다.

***

### 제한·보충사항 <a href="#a04" id="a04"></a>

* 사용자 페이지의 사용자 ID와 비밀번호로는 관리자 페이지에 로그인할 수 없습니다.
* SSO 연동을 통해 가져온 IdP 사용자는 ID와 비밀번호로 로그인할 수 없습니다. 다만, SSO 연동한 사용자 ID로 관리자 페이지에 접근하는 방법의 절차에 따라 관리자 페이지에 접근할 수는 있습니다.

***

### 절차 <a href="#a05" id="a05"></a>

{% hint style="warning" %}
**로그인을 클릭해도 반응이 없음**
{% endhint %}

DirectCloud에서 로그아웃하고 다음과 같은 방법으로 web 브라우저의 캐시와 쿠키를 모두 삭제합니다. 사용 중인 기기를 다시 시작하여 문제가 해결되는지 확인하세요.



**\[Google 크롬을 사용하는 경우]**

1. 다음 방법 중 하나를 사용하여 인터넷 사용 기록 데이터 지우기 화면을 표시합니다.

* Chrome 화면 오른쪽 상단의 '…'를 클릭하고 '설정' > '개인정보 보호 및 보안' > '인터넷 사용 기록 삭제'를 선택합니다.
* 주소 표시줄에 'chrome://settings/clearBrowserData' 입력 합니다.
* Ctrl+Shift+Del 키를 입력합니다.

2. '기간'에서 '모든 기간'을 선택하고 '캐시된 이미지 및 파일'과 '쿠키 및 기타 사이트 데이터'에만 체크를 한 다음 '데이터 삭제' 버튼을 클릭합니다.



**\[Microsoft Edge를 사용하는 경우]**

1. 다음 방법 중 하나를 사용하여 인터넷 사용 기록 데이터 지우기 화면을 표시합니다.

* Edge 브라우저의오른쪽 상단의 '…'를 클릭하고 '설정' > '개인정보, 검색 및 서비스' > '검색 데이터 지우기'를 선택합니다.
* 주소 표시줄에 'edge://settings/clearBrowserData' 입력합니다.
* Edge 화면에서 Ctrl+Shift+Del 키를 입력합니다.

2. '시간 범위'에서 '모든 기간'을 선택하고 '쿠키 및 기타 사이트 데이터'와 '캐시된 이미지 및 파일'만 체크하고 '지금 지우기' 버튼을 클릭합니다.



{% hint style="warning" %}
**로그인을 클릭하면 "아이디 또는 비밀번호가 일치하지 않습니다."라는 메시지가 표시됩니다.**
{% endhint %}

1. 관리자의 아이디와 비밀번호로 사용자 페이지에 로그인합니다.

<figure><img src="../../.gitbook/assets/image (700).png" alt=""><figcaption></figcaption></figure>



2. 사용자 페이지에서 암호를 변경하는 방법의 단계에 따라 암호를 변경합니다.



3. 다시 관리자자 페이지에 로그인합니다.

<figure><img src="../../.gitbook/assets/image (701).png" alt=""><figcaption></figcaption></figure>



{% hint style="warning" %}
**'아이디 또는 비밀번호가 일치하지 않습니다.' 이외의 오류 메시지가 표시됨**
{% endhint %}

1. 표시된 오류 메시지의 스크린샷을 저장합니다.



2. 다이렉트클라우드의 고객 지원팀에게 문의합니다.
