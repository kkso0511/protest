# 회사 휴지통에 있는 폴더, 파일을 복원하는 방법

### 개요 <a href="#a03" id="a03"></a>

이 매뉴얼에서는 '용량 관리' > '회사 휴지통'으로 이동 한 폴더, 파일을 사용자의 휴지통으로 복원하는 방법에 대해 설명합니다.

***

### 제한·보충사항 <a href="#a04" id="a04"></a>

* 관리자 권한이 부여된 사용자는 관리자 페이지에서 '용량 관리' 항목을 사용할 수 있습니다.
* My Box, Shared Box, Connect 폴더, DLP 폴더, 문서 관리 폴더, Warm Storage의 폴더, Cold Storage의 폴더로는, 직접 폴더와 파일을 복원할 수 없습니다.
* 회사 휴지통에서 삭제된 폴더·파일은 복원할 수 없습니다.
* 파일이 회사 휴지통으로 이동하더라도, 이전 버전은 유지됩니다. 회사 휴지통에서 파일·폴더를 복원하면, 이전 버전도 모두 복원됩니다.
* Connect User의 My Box는 다른 회사의 회사 ID로 관리되기 때문에, Connect User를 삭제하더라도 '회사 휴지통' 메뉴로 이동하지 않습니다.
* 자사에서 생성한 Connect 폴더에 저장된 파일을 다른 회사의 Connect User가 삭제한 경우에는, Connect User의 휴지통 'Connect'로 이동합니다.\
  그 상태에서 Connect User를 삭제하면, '회사 휴지통' 메뉴로 이동합니다.
* 'DirectCloud-CONNECT'를 이용할 수 있는 경우, 관리 페이지에 '공유 설정' > 'Connect 관리' 메뉴가 표시됩니다.&#x20;
* Connect User의 My Box는 다른 회사의 회사 ID로 관리되기 때문에, Connect User를 삭제하더라도 '회사 휴지통' 메뉴로 이동하지 않습니다.
* 자사에서 생성한 Connect 폴더에 저장된 파일을 다른 회사의 Connect User가 삭제한 경우에는, Connect User의 휴지통 'Connect'로 이동합니다.\
  그 상태에서 Connect User를 삭제하면, '회사 휴지통' 메뉴로 이동합니다.
* 'DirectCloud-SHIELD DLP'를 계약하고 있는 경우, 관리 페이지에 '공유 설정' > 'DLP 관리' 메뉴가 표시됩니다.
* Warm Storage를 이용할 수 있는경우, 관리 페이지에 '공유 설정' > 'Warm Storage' 메뉴가 표시됩니다.
* 'Cold Storage'를 계약하고 있는 경우에는, 관리 페이지에 '공유 설정' > 'Cold Storage 관리' 메뉴가 표시됩니다. Cold Storage에는 삭제 제한이 있기 때문에, 업로드, 이동, 자동 이동된 이후 90일 동안은 파일을 삭제할 수 없습니다.

***

### 절차 <a href="#a05" id="a05"></a>

{% hint style="warning" %}
**사용자가 삭제한 폴더 및 파일 복원**
{% endhint %}

사용자가 삭제하고 '회사 휴지통'으로 이동한 폴더, 파일을 복원하는 방법을 설명합니다.

1. '스토리지' > '회사 휴지통' 메뉴를 선택합니다.

<figure><img src="../../.gitbook/assets/image (2120).png" alt=""><figcaption></figcaption></figure>



2. 복원할 파일의 작업 열에서 복구 버튼을 클릭합니다.

<figure><img src="../../.gitbook/assets/image (2121).png" alt=""><figcaption></figcaption></figure>



3. 확인 화면에서 확인 버튼을 클릭합니다.

<figure><img src="../../.gitbook/assets/image (2122).png" alt=""><figcaption></figcaption></figure>



**\[사용자 페이지의 휴지통 확인]**

1. 사용자 페이지의 휴지통 메뉴에서 복원된 폴더 또는 파일을 확인합니다.

<figure><img src="../../.gitbook/assets/image (2123).png" alt=""><figcaption></figcaption></figure>



{% hint style="warning" %}
**삭제된 사용자 또는 Guest의 폴더 및 파일 복원**
{% endhint %}

삭제된 사용자 혹은 Guest의 폴더, 파일을 복원하는 방법에 대해 설명합니다. 복원 대상 사용자가 삭제되었으므로 복원할 소유자를 지정해야 합니다.

1. '용량 관리' > '회사 휴지통' 메뉴를 선택합니다.

<figure><img src="../../.gitbook/assets/image (2124).png" alt=""><figcaption></figcaption></figure>



2. '사용자' 열이 'i'(인포메이션 마크)로 표시된 행의 '복구' 버튼을 클릭합니다.\
   소유자 선택 화면이 표시됩니다.

<figure><img src="../../.gitbook/assets/image (2125).png" alt=""><figcaption></figcaption></figure>

&#x20;   &#x20;

3. 소유자 선택 화면에서 복원 후 소유자로 지정할 사용자 이름 또는 사용자 ID를 입력하고 검색 버튼을 클릭합니다.

<figure><img src="../../.gitbook/assets/image (2126).png" alt=""><figcaption></figcaption></figure>



4. 표시된 사용자의 선택 버튼을 클릭합니다.

<figure><img src="../../.gitbook/assets/image (2127).png" alt=""><figcaption></figcaption></figure>



5. 확인 화면에서 확인 버튼을 클릭합니다.

<figure><img src="../../.gitbook/assets/image (2128).png" alt=""><figcaption></figcaption></figure>



**\[사용자 페이지의 휴지통 확인]**

1. 사용자 페이지의 휴지통 메뉴에서 복원된 폴더 또는 파일을 확인합니다.

<figure><img src="../../.gitbook/assets/image (2129).png" alt=""><figcaption></figcaption></figure>
