# 모바일 애플리케이션에서 이미지를 저장할 수 있도록 설정하는 방법

### 개요 <a href="#a03" id="a03"></a>

이 매뉴얼에서는 ‘모바일 보안’ 메뉴에서 자사의 보안 정책에 따라 모바일 애플리케이션(iOS, Android)에서 DirectCloud의 이미지 파일을 저장할 수 있도록 할지 여부를 설정하는 방법에 대해 설명합니다.

***

### 제한·보충사항 <a href="#a04" id="a04"></a>

* 관리자 권한이 부여된 사용자는 관리자 페이지에서 '보안 정책'에 접근 가능합니다.
* ‘모바일 앱 보안 설정’은 Guest, LDAP 연동 및 SAML 연동으로 등록된 사용자에게도 적용되지만, Connect User에는 적용되지 않습니다.
* ‘이미지 다운로드’ 설정은 기본적으로 ‘사용 안함’으로 설정되어 있으므로, 모바일 애플리케이션에서 DirectCloud의 이미지 파일을 저장할 수 없습니다.
* ‘워크플로우’ > ‘기본 설정’ 메뉴에서 ‘다운로드’에 체크하면, ‘보안’ > ‘모바일’ 메뉴의 ‘이미지 저장’, ‘메일로 전송’, ‘다음 방법으로 열기’, ‘앱에서 열기’, ‘오프라인 시 접근’ 설정이 모두 ‘사용 안함’으로 고정되며, 모바일 애플리케이션에서 ‘⁝’ 버튼을 탭했을 때 ‘이미지 저장’, ‘다운로드’, ‘오프라인에서 사용 가능’ 메뉴가 표시되지 않게 됩니다.\
  다운로드 워크플로우를 ‘사용’에서 ‘사용 안함’으로 되돌리더라도, ‘이미지 저장’, ‘메일로 전송’, ‘다음 방법으로 열기’, ‘앱에서 열기’, ‘오프라인 시 접근’ 설정은 ‘사용 안함’으로 선택된 상태가 유지됩니다.

***

### 절차 <a href="#a05" id="a05"></a>

1. '보안 정책' > '모바일 보안' 항목을 선택합니다.
