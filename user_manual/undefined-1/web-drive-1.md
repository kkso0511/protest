# \[Web, Drive] 폴더를 이동하는 방법

### 개요 <a href="#a03" id="a03"></a>

이 매뉴얼에서는 Web 브라우저나 DirectCloud 드라이브를 사용하여 DirectCloud에 저장된 폴더를DirectCloud의 다른 폴더로 이동하는 방법에 대해 설명합니다.

***

### 동작 조건 <a href="#a03" id="a03"></a>

*   폴더 이동은 내부적으로 이동 대상 폴더가 삭제된 뒤, 이동 대상 폴더가 이동할 위치의 폴더로 업로드되는 방식으로 처리됩니다.\
    따라서 폴더를 이동하는 사용자에게는 이동 원본 폴더와 이동 대상 폴더 모두에 아래의 접근 권한이 부여되어 있어야 합니다.

    * **이동 원본 폴더:**\
      마스터, 전체권한
    * **이동 대상 폴더:**\
      마스터, 전체권한, 편집자, 편집자－, 프리뷰어＋, 업로더

    자세한 내용은 [접근 권한의 종류와 Web 브라우저에서 가능한 조작](https://help.directcloud.net/user_manual/undefined-5/web), [접근 권한의 종류와 DirectCloud 드라이브 에서 가능한 조작](https://help.directcloud.net/user_manual/undefined-5/directcloud)을 참조하십시오.
* 이동 화면에서 이동 대상 폴더에 새 폴더를 생성하려면 <마스터>, <전체권한> 접근 권한이 부여되어 있어야 합니다.
* 다음 조건을 모두 만족하는 경우, DirectCloud 드라이브에서는 폴더를 복사할 수 없습니다.
  1. [워크플로우를 활성화하는 방법](https://help.directcloud.net/admin_manual/undefined-7/undefined)에서 워크플로우가 활성화되어 있음
  2. [특정 폴더에서 워크플로우를 활성화하는 방법](https://help.directcloud.net/admin_manual/undefined-4/undefined-20)에서 복사하려는 폴더가 포함된 폴더 또는 복사 대상 폴더의 워크플로우가 사용하기로 설정되어 있음
  3. 다운로드 워크플로우 또는 업로드 워크플로우의 신청자로 설정되어 있음
* Warm Storage로 이동을 지원하는 Hot Storage는 Shared Box만 해당됩니다. \
  Warm Storage와 관련된 폴더 및 파일 작업에 대해서는[【Web, Drive】 Warm Storage로 폴더 또는 파일을 이동하는 방법](https://help.directcloud.net/user_manual/warm-storage/web-drive-warm-storage) 및 [사용자 페이지에서 고스트(Ghost)를 조작하는 방법](https://help.directcloud.net/user_manual/warm-storage/ghost) 을 참고해 주시기 바랍니다.
*   Cold Storage에서는 아래 방향으로 폴더 및 파일을 이동할 수 있습니다.<br>

    Hot Storage → Cold Storage\
    Cold Storage → Cold Storage\
    Warm Storage → Cold Storage<br>

    단, Cold Storage로 이동을 지원하는 Hot Storage는 Shared Box만 해당됩니다.

***

### 제한·보충사항 <a href="#a04" id="a04"></a>

* 사용자 A를 포함한 여러 사용자에게 <편집자>의 공유자 권한이 부여된 폴더에서 사용자A에만 액세스 수준이 부여된 폴더로 폴더를 이동한 경우 대상 폴더에서 사용자 A 이외의 액세스 권한이 빠집니다.\
  이후, 대상 폴더에 파일이 업로드되어도 사용자 A 이외에 업로드 알림 메일이 도착하지 않습니다. 업로드 알림 이메일에 대한 자세한 내용은 지정된 폴더에서 업로드 알림을 사용하는 방법을 참조하세요.
* 여러개의 폴더를 체크하여 선택한 경우 위에서 순서대로 이동이 수행됩니다.
* 대상 폴더에 이름이 같은 폴더가 있으면 이동할 수 없습니다.
* 작성된 링크가 있는 파일이 들어 있는 폴더를 복사하면 파일의 링크 정보는 복사되지 않지만 폴더를 이동하면 파일의 링크 정보가 유지됩니다.
* DirectCloud 드라이브에서 로컬 환경의 폴더로 폴더를 이동하면 이동하는 대신 '복사'를 처리합니다.
* 여러 폴더와 파일을 선택하고 이동할 때 한 번에 이동할 수 있는 하위 폴더 수는 총 10,000개입니다.\
  10,001개 이상의 하위 폴더가 있는 상위 폴더를 이동하려고 하면 "복사 또는 이동 대상이 최대값(10,000개)을 초과하므로 작업을 수행할 수 없습니다."라는 오류 메시지가 표시됩니다.\
  이 경우 이동 처리 자체가 중단됩니다.
* 대량의 폴더 및 파일을 이동하는 경우 로드로 인해 처리 속도가 느려질 수 있으므로 DirectCloud에서는 한 번에 1,000개 이상의 폴더 및 파일을 이동하는 것을 권장하지 않습니다. 특히 다음에 해당하는 경우는 주의해 주십시오.
  * 수십만 개의 버전 정보가 있는 파일 또는 파일이 저장된 폴더를 이동하는 경우
  * 'Shared Box'에서 'My Box' 등 큰 범위로 이동하는 경우
* DirectCloud 드라이브에서 많은 양의 폴더와 파일을 이동하는 경우 대상 폴더에 반영될 때까지 폴더와 파일이 제대로 표시되지 않으므로 다음 사항에 유의하십시오.
  * 이동 처리가 완료될 때까지 처리 대상의 폴더나 파일의 조작을 하지 않는다
  * 대상 폴더에서 이동한 파일이 사라진 것처럼 보이는 경우 몇 시간 정도 경과한 후 다시 확인합니다.
  * 대상 폴더에 파일이 없으면 원본 폴더에 파일이 남아 있는지 확인하고 한 번에 이동하는 파일 수를 줄이고 다시 이동합니다.
* 이동이 진행되는 동안 상태 화면에서 '×'를 클릭하거나 세션 시간의 10분을 초과하면 '복사/이동 중 타임아웃이 발생하여 더 이상 진행할 수 없습니다.'라는 오류 메시지가 표시됩니다.  \
  이 경우 오류 메시지가 나타날 때까지 처리 된 폴더와 대상 폴더에 저장된 폴더 파일은 이동 되지만 처리 대기중인 폴더 파일은 이동 되지 않습니다.
* DirectCloud AI가 활성화된 폴더에서 폴더 이동 범위는 다음과 같습니다.
  * DirectCloud AI 사용 가능 폴더로 이동할 수 있는 폴더 \
    ⇒ My Box, Shared Box
  * DirectCloud AI 사용 가능 폴더에서 이동할 수 있는 폴더 \
    ⇒ My Box, Shared Box, Warm Storage, Cold Storage
* DirectCloud AI 설정이 사용 가능한 폴더를 이동하면 벡터 데이터는 다음과 같이 처리됩니다.
  *   공유 메뉴에서 DirectCloud AI 설정이 잘못된 폴더로 이동:

      폴더 속성 설정은 유지되므로 폴더 내에서 생성된 벡터 데이터는 유지됩니다.
  *   공유 메뉴 이외의 폴더로 이동:

      DirectCloud AI 설정 정보가 삭제되고 벡터 데이터도 삭제됩니다.

      폴더를 다시 공유 메뉴로 이동하면 DirectCloud AI 설정이 표준 사용 안 함으로 돌아갑니다. 삭제된 벡터 데이터를 바탕으로 되돌릴 수 없다는 점에 주의가 필요합니다.
*   Cold Storage로 폴더나 파일을 이동하면 다음 정보가 삭제됩니다.

    * 폴더 액세스 권한
    * 폴더 속성
    * 링크 내역
    * 파일 잠금
    * 파일 마감
    * 즐겨찾기 정보
    * 전체 텍스트 검색 데이터

    다음 정보는 보존됩니다.

    * 태그
    * 버전 기록
    * 코멘트
    * 액세스 내역
    * DirectCloud-SHIELD IRM으로 암호화된 파일

***

### 절차 <a href="#a05" id="a05"></a>

{% hint style="warning" %}
**Web 브라우저에서 폴더 이동**
{% endhint %}

**\[조작 메뉴에서 폴더 이동]**

1. 이동하려는 폴더가 저장된 폴더를 클릭합니다.
2. 다음 방법 중 하나로 폴더의 이동 화면을 표시합니다.\
   ❶ 이동하고자 하는 폴더를 체크하고 상단의 '이동' 버튼을 클릭합니다. 여러 폴더를 선택할 수 있습니다.\
   ❷  이동하려는 폴더를 체크하고 마우스 오른쪽 버튼으로 클릭하고 표시된 메뉴에서 \[이동]을 클릭합니다. 여러 폴더를 선택할 수 있습니다.\
   ❸  이동할 폴더의 오른쪽에 있는 ▼ 버튼을 클릭한 다음 표시된 메뉴에서 이동을 클릭합니다. 여러 폴더를 선택할 수 없습니다.\
   \
   이동 화면이 표시됩니다.
3. 이동 화면에서 이동 폴더를 클릭합니다.
4. 필요한 경우 새 폴더를 클릭하여 새 폴더를 만듭니다.
5. 이동 버튼을 클릭합니다.\
   이동이 시작되고 상태 화면이 표시됩니다.\
   이동이 완료되면 화면 하단에 "이동이 완료되었습니다."라는 메시지가 표시됩니다.\
   메시지에 표시되는 취소 버튼을 클릭하여 이동을 취소할 수 있습니다.

\
**\[드래그 앤 드롭으로 폴더 이동]**

1. 이동하려는 폴더가 저장된 폴더를 클릭합니다.
2.  이동하려는 폴더를 드래그 하여 대상 폴더에 놓습니다.

    * 여러 폴더를 체크하고 드래그 앤 드롭 할 수도 있습니다.
    * "선택한 파일/폴더를 드롭한 폴더로 이동하시겠습니까?"라는 확인 화면이 표시되면 "확인" 버튼을 클릭하십시오.\
      \
      이동이 시작되고 상태 화면이 표시됩니다.\
      이동이 완료되면 화면 하단에 "이동이 완료되었습니다."라는 메시지가 표시됩니다.\
      메시지에 표시되는 취소 버튼을 클릭하여 이동을 취소할 수 있습니다.



{% hint style="warning" %}
**DirectCloud 드라이브에서 폴더 이동**
{% endhint %}

1. DirectCloud Drive에서 이동하려는 폴더를 대상 폴더로 끌어다 놓습니다.\
   여러 폴더와 파일을 선택하여 이동할 수도 있습니다.
2. 폴더가 이동되었는지 확인합니다.
