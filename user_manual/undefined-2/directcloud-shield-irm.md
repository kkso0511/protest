# 사용자 페이지의 특정 폴더에서 DirectCloud-SHIELD IRM을 활성화하는 방법

### 개요 <a href="#a03" id="a03"></a>

이 매뉴얼 에서는 사용자 페이지의 특정 폴더에서 DirectCloud-SHIELD IRM 암호화를 활성화하는 방법을 설명합니다.

### 제한·보충사항 <a href="#a04" id="a04"></a>

* 폴더 속성을 설정하려는 사용자에게는 '마스터' 접근 권한이 부여되어 있어야 합니다.\
  자세한 내용은 [접근 권한의 종류와 Web 브라우저에서 가능한 조작](https://help.directcloud.net/user_manual/undefined-5/web)을 참조해주시기 바랍니다.
* 초기 상태에서는 SHIELD가 사용하지 않음으로 설정되어 있습니다.
* SHIELD는 1단계 폴더에만 설정할 수 있으며, 설정 내용은 하위 폴더에 자동으로 이어집니다.
* 관리자가 DirectCloud-SHIELD IRM을 활성화하면 폴더 속성에 SHIELD가 표시됩니다.\
  단, DLP 기능을 사용 중인 경우 이 메뉴는 표시되지 않습니다.
* DirectCloud-SHIELD DLP를 계약한 경우 사용자 페이지에 DLP 메뉴가 표시됩니다.
* DirectCloud-CONNECT를 이용할 수 있는 경우 사용자 페이지에 Connect 메뉴가 표시됩니다.
* Connect 폴더의 폴더 속성에서는 워크플로우, SHIELD, 링크를 설정할 수 없습니다.
* Warm Storage를 계약한 경우 사용자 페이지에 Warm Storage 메뉴가 표시됩니다.\
  Warm Storage에서는 DirectCloud-SHIELD IRM을 사용할 수 없으므로, Warm Storage의 폴더 속성에는 SHIELD가 표시되지 않습니다.

### 절차

1. Shared Box 메뉴를 클릭한 뒤, DirectCloud-SHIELD IRM을 설정하려는 1단계 폴더를 클릭합니다.

<figure><img src="../../.gitbook/assets/image (2408).png" alt=""><figcaption></figcaption></figure>

2. 슬라이드 화면의 폴더 속성 설정에서 SHIELD가 사용함으로 표시되어 있는지 확인합니다.\
   '사용 안함'으로 표시되어 있는 경우, 오른쪽에 표시된 설정 버튼을 클릭합니다.

<figure><img src="../../.gitbook/assets/image (2409).png" alt=""><figcaption></figcaption></figure>

3. SHIELD 설정 화면이 표시됩니다.\
   '사용'을 선택한 뒤 확인 버튼을 클릭합니다.
