# 사용자 페이지에서 파일이 Warm Storage로 자동 이동되도록 설정하는 방법

### 개요 <a href="#a03" id="a03"></a>

DirectCloud는 Shared Box에서 사용하는 스토리지(Hot Storage) 외에 'Warm Storage'를 제공합니다. 일반 Hot Storage에서 링크 및 버전 관리와 같은 일부 기능을 제외한 Warm Storage는 저렴한 가격으로 사용할 수 있습니다. 예를 들어, 자주 사용하지 않는 폴더의 데이터를 Warm Storage에 장기간 저장하면 Shared Box의용량을 효과적으로 사용할 수 있습니다.\
이 매뉴얼에서는 사용자 페이지에서 Shared Box의 폴더 속성을 표시하고 파일이 Warm Storage로 자동 이동되도록 설정하는 방법에 대해 설명합니다.

***

### 제한·보충사항 <a href="#a04" id="a04"></a>

* 폴더 속성을 설정하는 사용자에게는 <마스터>의 접근 레벨이 부여되어 있어야 합니다.
* Warm Storage를 사용할 수 있다면 사용자 페이지에 ‘Warm Storage’ 메뉴가 표시됩니다. 또한, 폴더 속성에 ‘자동 이동’ 설정이 표시됩니다.
* 자동 이동 대상으로 설정할 수 있는 것은 1단계부터 6단계까지의 Shared Box 내 하위 폴더만 해당됩니다.
* 상위 폴더에 자동 이동이 설정되어 있는 경우, 하위 폴더에는 그 설정이 상속됩니다.\
  이처럼 자동 이동 설정이 상속된 하위 폴더의 폴더 속성에서는 ‘자동 이동’ 설정에 ‘변경’ 버튼이 표시되지 않습니다.
* 또한, 상위 폴더의 설정에 따라 자동 이동된 하위 폴더 내 파일에는 고스트(Ghost)가 표시됩니다.
* 반대로, 하위 폴더에 자동 이동이 설정되어 있는 경우, 상위 폴더에서는 자동 이동을 ‘사용함’으로 설정할 수 없으므로, 폴더 속성의 ‘자동 이동’ 설정에 ‘설정’ 버튼이 표시되지 않습니다.
* 자동 이동은 폴더 단위로 설정합니다.\
  예를 들어, 아래와 같은 구성의 Shared Box에서 폴더 C에 대해 Warm Storage 폴더 A로의 자동 이동을 설정하면, Warm Storage 폴더 A에 폴더 C 이하의 폴더가 계층 구조를 유지한 상태로 생성되며, 폴더 C 이하의 파일이 이동됩니다.
* 자동 이동과 관련된 로그는 파일의 ‘파일 사용 이력’ 화면에 아래와 같이 기록됩니다.\
  \- 파일이 Warm Storage로 자동 이동된 경우: ‘자동 이동’\
  \- Warm Storage에서 원래 폴더로 파일을 되돌린 경우: ‘이동’
* 파일의 자동 이동 대상 위치에 이미 동일한 이름의 폴더가 있는 경우, 해당 폴더로 파일이 자동 이동됩니다.
* 자동 이동 원본 폴더의 이름이 변경된 경우, 자동 이동이 실행될 때 변경된 이름으로 Warm Storage에 새로운 폴더가 생성되고 파일이 자동 이동됩니다.
* 자동 이동 대상 위치에 동일한 이름의 파일이 있는 경우, 파일 이름 끝에 숫자가 추가됩니다. 이동 원본의 고스트(Ghost) 이름 끝에도 동일하게 숫자가 추가됩니다.
* 자동 이동 대상으로 설정된 Warm Storage 폴더의 이름이나 폴더 경로가 변경된 경우, ‘자동 이동’ 설정의 ‘이동 대상’ 정보도 함께 변경됩니다.
* 자동 이동 대상으로 설정된 Warm Storage 폴더가 삭제된 경우, 자동 이동이 실행될 때 해당 폴더가 자동으로 새로 생성됩니다.
* 파일이 자동 이동되면 다음 정보가 삭제됩니다.
  * 링크 이력
  * 파일 잠금
  * 파일 유효기간
* 최종 파일 사용 날짜가 파일 자동 이동의 기준이 됩니다.\
  최종 파일 사용 날짜의 다음 날을 기산일로 하여, 기산일을 포함해 본 문서의 절차에서 설정한 일수가 경과한 후, 자정 0시부터 순차적으로 파일이 자동 이동됩니다.
*   다음 작업 중 하나라도 수행되면, 대상 폴더에 저장된 파일의 파일 사용 날짜가 갱신됩니다.

    * 업로드
    * 덮어쓰기 저장
    * 이름 변경
    * 이동
    * 자동 이동
    * 복사
    * Cold Storage에서 복원
    * 이전 버전으로 복원
    * 휴지통에서 복원
    * 삭제
    * 자동 삭제(파일 유효기간)
    * 자동 삭제(My Box)
    * 자동 삭제(폴더 속성)
    * 휴지통 이동
    * 이동 취소
    * 삭제 취소
    * 휴지통 취소
    * 온라인 편집
    * 프리뷰
    * DirectCloud Drive에서 열기
    * 오프라인 저장
    * 다운로드
    * 첨부파일로 전송
    * 다른 모바일 메일 앱에서 전송
    * 다른 모바일 앱에서 전송
    * 링크 생성

    이러한 작업의 로그는 파일의 ‘파일 사용 이력’ 화면에서 확인할 수 있습니다.

***

### 절차 <a href="#a05" id="a05"></a>

{% hint style="warning" %}
**자동 이동 설정**
{% endhint %}

1. Shared Box 메뉴를 클릭하고 자동 이동을 설정할 폴더를 클릭합니다.

<figure><img src="../../.gitbook/assets/스크린샷 2025-10-22 172410.png" alt=""><figcaption></figcaption></figure>



2. 슬라이드 화면의 '폴더 속성' 설정에서 '자동이동' 설정을 확인합니다.\
   자동 이동 설정이 되어있지 않으면 '사용안함'이라고 표시됩니다.\
   이미 자동 이동을 설정한 경우 '사용함'이 표시됩니다.

<figure><img src="../../.gitbook/assets/스크린샷 2025-10-22 172652.png" alt=""><figcaption></figcaption></figure>



3. 자동 이동 설정 오른쪽에 표시된 설정 버튼을 클릭합니다.

<figure><img src="../../.gitbook/assets/스크린샷 2025-10-22 173134.png" alt=""><figcaption></figcaption></figure>

'자동 이동' 설정이 표시됩니다.



4. 자동 이동을 위해 '사용함'을 선택한 다음 '이동 위치'를을 클릭합니다.

<figure><img src="../../.gitbook/assets/스크린샷 2025-10-22 173246.png" alt=""><figcaption></figcaption></figure>



5. 파일을 이동할 폴더를 선택하고 OK 버튼을 클릭합니다.

<figure><img src="../../.gitbook/assets/스크린샷 2025-10-22 173337.png" alt=""><figcaption></figcaption></figure>



6. 드롭다운 목록에서 일 수를 선택하면 마지막으로 액세스한 날짜 이후의 파일로 자동으로 이동합니다. '10 \~ 2000'의 숫자를 선택할 수 있습니다.

<figure><img src="../../.gitbook/assets/스크린샷 2025-10-22 173406 (1).png" alt=""><figcaption></figcaption></figure>



7. 확인 버튼을 클릭합니다.\
   '폴더 속성' 설정에서 '자동 이동' 설정을 표시합니다.   \
   끝에 있는 "i" 아이콘 위로 마우스를 가져가면 툴팁을 사용하여 이동하려는 폴더를 확인할 수 있습니다.

<figure><img src="../../.gitbook/assets/스크린샷 2025-10-22 173443.png" alt=""><figcaption></figcaption></figure>



{% hint style="warning" %}
**자동 이동 완료 메일 확인**
{% endhint %}

자동 이동으로 설정된 폴더 아래의 파일이 자동으로 이동되면 원래 폴더에 액세스할 수 있는 모든 사용자에게 "\[DirectCloud] Shared Box 자동이동 안내"라는 제목의 이메일이 전송됩니다.<br>
