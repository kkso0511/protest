# 접근 권한의 종류와 PC Agent에서 가능한 조작

### 개요 <a href="#a03" id="a03"></a>

DirectCloud에서는 부여된 접근 권한에 따라 사용자 또는 그룹이 폴더와 파일에 대해 수행할 수 있는 작업이 다릅니다.\
이 매뉴얼에서는 접근 권한의 종류와 PC Agent 에서 수행할 수 있는 작업에 대해 설명합니다.

***

### 제한·보충사항 <a href="#a04" id="a04"></a>

* 최상위 계층 폴더에 대한 작업은 Web 브라우저에서만 가능합니다.
* PC  Agent 에서는 다음과 같은 설정 및 작업을 수행할 수 없습니다.
  * 미리보기
  * 파일 코멘트
  * 온라인 편집
  * 파일 유효기간 설정
  * 파일첨부 메일발송
  * 업로드 요청
  * 파일 잠금
  * 태그
  * 파일 사용 이력
  * Guest 초대
  * Guest 초대 승인
  * Connect User 초대
  * 접근 권한 설정
  * 폴더 속성 설정
* Guest에게 부여할 수 있는 접근 권한은 사용자 및 그룹과 동일합니다.\
  단, Guest는 ‘첨부파일 전송’, ‘업로드 요청’, ‘Guest 초대’, ‘Guest 초대 승인’, ‘접근 권한 설정’, ‘폴더 속성 설정’을 이용할 수 없습니다.
* Guest는 ‘파일 링크 만들기’와 ‘폴더 링크 만들기’를 사용할 수 있지만, 관리자가해당 기능을 제한할 수 있습니다.
* 또한 Guest는 최상위 계층의 폴더를 조작할 수 없으므로, Guest에게 <마스터> 권한을 부여할 수 없습니다.
* Connect 폴더에서는 ‘파일 링크 만들기’, ‘폴더 링크 만들기’, ‘첨부파일 전송’, ‘Guest 초대’를 사용할 수 없습니다.

***

### 설명 <a href="#a05" id="a05"></a>

사용자별로 별도의 권한을 설정할 수 있습니다. 각 권한별 제공기능은 아래 표와 같습니다.

<figure><img src="../../.gitbook/assets/image (473).png" alt=""><figcaption></figcaption></figure>

* 그룹권한과 사용자권한을 동시에 받아 복수권한인 경우 합집합으로 제공됩니다.
* Guest 사용자는 전체권한이라도 'Guest 초대' 기능을 사용할 수 없습니다.
* Guest 사용자는 전체권한, 편집자, 편집자- 라도 '업로드 요청', '파일첨부 메일발송' 기능을 사용할 수 없습니다.
* Guest 사용자는 관리자의 기능 제한 설정에 따라'링크' 기능을 사용할 수 없습니다.
* '편집자'의 파일삭제 권한은 본인이 업로드한 파일로 제한됩니다.
* '마스터'는 일부 권한이 제한될 수 있으며, 각 마스터별 관리범위는 별도로 확인할 수 있습니다.
* ※ 폴더별 설정에 따라'권한 관리'를 전체권한에게 부여할 수 있습니다. 단, Guest는 전체권한이라도 해당 기능을 사용할 수 없습니다.
