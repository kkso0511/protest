# \[Web, Drive] 링크를 메일프로그램으로 보내는 방법

### 개요 <a href="#a03" id="a03"></a>

이 매뉴얼에서는 사용 중인 기기에 설치된 메일 소프트웨어를 통해 링크 정보를 보내는 방법에 대해 설명합니다.

***

### 제한·보충사항 <a href="#a04" id="a04"></a>

* 외부 메일 프로그램 열기 기능을 통해 링크 정보를 전송할 수 있는 메일 프로그램은 Outlook과 Thunderbird만 지원됩니다.
* 그 외의 메일 프로그램으로 전송하려는 경우에는 [\[Web\] 링크 생성 화면을 표시하여 기본 기능을 확인하는 방법](https://help.directcloud.net/user_manual/undefined-4/web) 또는 [\[Dirve\] 링크 생성 화면을 표시하여 기본 기능을 확인하는 방법](https://help.directcloud.net/user_manual/undefined-4/dirve)을 참고하여 링크 복사 아이콘으로 링크를 복사한 뒤 메일 프로그램에 붙여넣어 사용해주시기 바랍니다.
* 사용자 로그인 페이지의 메인 도메인을 설정하는 방법의 절차에서 링크 URL의 도메인을 설정하면, 생성되는 링크 URL의 메인 도메인도 함께 변경됩니다.
* 메인 도메인을 변경하더라도 변경 이전의 Shared Box 링크에는 계속 접근할 수 있습니다.
*   링크의 접근 횟수는 세션 단위로 계산되며, 해당 정보는 링크를 연 Web 브라우저에 쿠키로 저장됩니다.\
    따라서 동일한 사용자가 같은 세션에서 10회 접근하더라도 접근 횟수는 1회로 취급됩니다.

    브라우저 방문 기록을 삭제하거나 브라우저를 연 후 24시간이 지나면 세션은 종료됩니다.
* 전달한 링크를 열 때 두 번째 이후에도 비밀번호를 다시 입력하도록 설정할 수는 없습니다.
* 공유 링크 화면에서 URL을 클릭해도 링크 수신 화면의 접근 횟수에는 추가되지 않습니다.\
  단, 아래의 경우에는 접근 횟수가 추가됩니다.
  * Google Chrome 이외의 브라우저에서 다른 탭으로 링크를 연 경우
  * 브라우저를 연 후 24시간이 지난 경우
* 링크 생성 후 대상 폴더 또는 파일이 저장된 폴더의 접근 권한이 삭제되면 링크 대상 폴더 및 파일에 접근할 수 없습니다.\
  또한 링크를 생성한 사용자의 접근 권한이 삭제되면 링크 수신자 역시 해당 폴더와 파일에 접근할 수 없습니다.
* 링크 수신 화면에서 파일을 미리보기 할 때 표시되는 날짜와 시간은 파일의 업데이트 시각이며, 이를 비표시로 설정할 수는 없습니다.\
  같은 파일명으로 덮어쓰기 저장하면 링크 수신 화면에 표시되는 파일 업데이트 시각도 변경됩니다.

***

### 절차 <a href="#a05" id="a05"></a>

{% hint style="warning" %}
**Web 브라우저에서 생성한 링크 정보를 메일 프로그램으로 전송하는 방법**
{% endhint %}

1. Web 브라우저에서 [링크 생성 화면을 표시하여 기본 기능을 확인하는 방법](https://help.directcloud.net/user_manual/undefined-4/web)을 참고하여 링크 생성 화면을 표시합니다.



2. 외부 메일 프로그램 열기 아이콘을 클릭합니다.

<figure><img src="../../.gitbook/assets/image (2445).png" alt=""><figcaption></figcaption></figure>

&#x20;      PC에 설치되어 있는 메일 프로그램이 실행되며, 메일 작성 화면에 링크 정보가 추가됩니다.

<figure><img src="../../.gitbook/assets/image (2447).png" alt=""><figcaption></figcaption></figure>



3. 받는 사람을 입력한 뒤 메일을 전송합니다.

1. DirectCloud 드라이브에서 [링크 생성 화면을 표시하여 기본 기능을 확인하는 방법](https://help.directcloud.net/user_manual/undefined-4/dirve)을 참고하여 링크 생성 화면을 표시합니다.
