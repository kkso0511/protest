# \[Web, Drive] 계정이 없는 상대로부터 파일을 받는 방법(업로드 요청 기능)

### 개요 <a href="#a03" id="a03"></a>

이 매뉴얼에서는 업로드 요청 기능을 사용하여, DirectCloud 계정을 보유하지 않은 사용자로부터 파일을 전달받는 방법을 설명합니다.

***

### 제한·보충사항 <a href="#a04" id="a04"></a>

* 이 매뉴얼의 절차는 Shared Box에서의 사용 방법을 기준으로 설명하고 있지만, DLP 폴더 에서도 동일한 방식으로 사용할 수 있습니다.
* 다음 폴더에서는 업로드 요청 기능을 사용할 수 없습니다.&#x20;
  * Connect 폴더
  * Warm Storage 폴더
* 업로드 요청 기능을 사용하려면, 관리자 페이지의 사용권한 설정에서 ‘File Request’ 항목에 체크되어 있는 권한 세트가 그룹 및 사용자에게 부여되어 있어야 합니다.
* 또한, 업로드 요청 기능을 사용하려면 <마스터>, <전체권한>, <편집자>, <편집자－> 접근 권한이 필요합니다.
* 폴더 자체를 업로드 요청 대상으로 지정할 수는 없습니다.
* '받는 사람', '제목', '내용'을 입력 한 후 '업로드 요청 링크 전송' 화면의 왼쪽 상단에 있는 '닫기' 버튼을 클릭하면 설정 내용이 임시 저장됩니다.
* 이후 동일한 폴더에서 업로드 요청 생성을 다시 시작하고 표시된 확인 화면에서 확인 버튼을 클릭하면, 이전에 임시 저장된 설정 내용을 불러올 수 있습니다.
* 업로드 요청 링크 전송 화면에 표시되는 발신자 이메일 주소는 no-reply 메일에 회신할 수 있도록 설정되어 있는지 여부에 따라 다릅니다.
  * no-reply 메일에 회신할 수 있는 경우: 사용자 이름 no-reply@directcloud.jp
  * no-reply 메일에 회신할 수 없는 경우: DirectCloud no-reply@directcloud.co.jp
* 사용자 페이지의 업로드 요청 이력 메뉴와 관리자 페이지의 '로그 현황'> '업로드 요청 내역' 메뉴에서는, 비활성화된 업로드 요청 링크를 다시 활성화할 수 없습니다.

**\[업로드 요청 링크 보내기]**

1. 업로드 요청 링크 보내기 화면에서 받는 사람에게 링크를 보낼 이메일 주소를 입력합니다.\
   입력란 이외의 장소를 클릭하면 입력한 이메일 주소가 반영됩니다.\
   [회사·개인 주소록을 활성화하는 방법](https://help.directcloud.net/admin_manual/undefined-6/undefined-8)에 따라 주소록이 활성화된 경우 오른쪽에 표시된 아이콘을 클릭하여 주소록에서 대상 이메일 주소를 선택할 수도 있습니다.&#x20;

<figure><img src="../../.gitbook/assets/image (1099).png" alt=""><figcaption></figcaption></figure>



2. 필요한 경우 제목과 내용을 입력합니다.

<figure><img src="../../.gitbook/assets/image (1100).png" alt=""><figcaption></figcaption></figure>



3. 전송되는 이메일의 내용을 확인하려면 미리보기 버튼을 클릭합니다.

<figure><img src="../../.gitbook/assets/image (1101).png" alt=""><figcaption></figcaption></figure>



4. 이메일 내용을 확인하고 ✕ 버튼을 클릭하여 미리보기를 닫습니다.

<figure><img src="../../.gitbook/assets/image (1102).png" alt=""><figcaption></figcaption></figure>



5. 설정한 내용으로 링크를 보낼 수 있는 경우에는 \[보내기] 버튼을 클릭합니다.

<figure><img src="../../.gitbook/assets/image (1103).png" alt=""><figcaption></figcaption></figure>

&#x20;     업로드 요청의 URL이 전송됩니다.

**\[업로드 요청 링크 보내기]**

1. 업로드 요청 링크 보내기 화면에서 받는 사람에게 링크를 보낼 이메일 주소를 입력합니다.\
   입력란 이외의 장소를 클릭하면 입력한 이메일 주소가 반영됩니다.\
   [회사·개인 주소록을 활성화하는 방법](https://help.directcloud.net/admin_manual/undefined-6/undefined-8)에 따라 주소록이 활성화된 경우 오른쪽에 표시된 아이콘을 클릭하여 주소록에서 대상 이메일 주소를 선택할 수도 있습니다.&#x20;

<figure><img src="../../.gitbook/assets/image (1106).png" alt=""><figcaption></figcaption></figure>



2. 필요한 경우 제목과 내용을 입력합니다.

<figure><img src="../../.gitbook/assets/image (1107).png" alt=""><figcaption></figcaption></figure>



3. 전송되는 이메일의 내용을 확인하려면 미리보기 버튼을 클릭합니다.

<figure><img src="../../.gitbook/assets/image (1108).png" alt=""><figcaption></figcaption></figure>



4. 이메일 내용을 확인하고 ✕ 버튼을 클릭하여 미리보기를 닫습니다.

<figure><img src="../../.gitbook/assets/image (1109).png" alt=""><figcaption></figcaption></figure>



5. 설정한 내용으로 링크를 보낼 수 있는 경우에는 \[보내기] 버튼을 클릭합니다.

<figure><img src="../../.gitbook/assets/image (1110).png" alt=""><figcaption></figcaption></figure>

&#x20;     업로드 요청의 URL이 전송됩니다.
