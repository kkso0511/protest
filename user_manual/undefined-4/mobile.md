# \[Mobile] 링크 생성 화면을 표시하여 기본 기능을 확인하는 방법

### 개요 <a href="#a03" id="a03"></a>

이 매뉴얼에서는 iOS 애플리케이션 또는 Android 애플리케이션에서 DirectCloud에 저장되어 있는 폴더·파일의 링크를 생성·전송하기 위한 '링크' 화면을 표시하고, 화면 구성과 기본적인 기능을 확인하는 방법에 대해 설명합니다.

***

### 제한·보충사항 <a href="#a04" id="a04"></a>

* 관리자 페이지의  '보안 정책'->'사용 권한' 에서 'File Link'에 체크가 되어 있는 권한 세트가 그룹 또는 사용자에게 부여되어 있어야 합니다.
* 링크 생성 화면을 표시하려면, [특정 폴더에서 링크 생성 기능을 활성화하는 방법](https://help.directcloud.net/admin_manual/undefined-4/undefined-23)의 절차에 따라 아래와 같이 설정해야 합니다.
  * Shared Box, DLP: '폴더 옵션' 설정에서 '링크'의 '사용'에 체크
* 폴더 및 파일의 공유 링크를 생성하려면, <마스터>, <전체권한>, <편집자> 접근 권한이 부여되어 있어야 합니다.
* "수신자", "참조／숨은참조", "제목", "내용"을 입력한 뒤, 링크 생성 화면의 오른쪽 상단에 있는 "✕" 버튼 또는 "취소" 버튼을 클릭하면 설정 내용이 임시 저장됩니다.
* 이후 동일한 폴더·파일의 링크 생성을 다시 시작하고 표시된 확인 화면에서 "확인" 버튼을 클릭하면, 임시 저장된 설정 내용을 불러와 표시할 수 있습니다.
* 링크의 접근 횟수는 세션 단위로 계산되며, 그 횟수는 링크를 연 웹 브라우저에 쿠키로 저장됩니다.\
  따라서 한 사용자가 동일한 세션 내에서 링크를 10회 열더라도, 접근 횟수는 1회로 계산됩니다.\
  브라우저의 탐색 기록을 삭제하거나 브라우저를 연 후 24시간이 지나면 해당 세션은 종료됩니다.
* 전송된 링크를 열 때, 두 번째 이후에도 비밀번호를 다시 입력하도록 설정할 수는 없습니다.
* 링크를 생성한 이후, 대상 폴더 또는 파일이 저장된 폴더에 대한 접근 권한이 삭제된 경우에는 링크 대상의 폴더·파일에 접근할 수 없습니다.\
  링크를 생성한 사용자의 접근 권한이 삭제되면, 링크 수신자 또한 해당 폴더·파일에 접근할 수 없게 됩니다.
* 링크 수신 화면에서 파일을 미리보기할 때 표시되는 날짜 및 시간은 파일의 업데이트 시각입니다. \
  이 시각을 비표시로 설정할 수는 없습니다.\
  동일한 파일명으로 덮어쓰기 저장하면, 링크 수신 화면의 파일 업데이트 시각도 변경됩니다.
* [폴더 용량 제한을 설정](https://help.directcloud.net/admin_manual/undefined-4/undefined-18)한 경우, 제한된 용량을 초과하는 파일은 링크를 통해 업로드할 수 없습니다.
* [업로드 가능한 파일의 크기를 제한하는 방법](https://help.directcloud.net/admin_manual/undefined/undefined-2)의 절차에서 파일 단위 용량 제한이 활성화되어 있는 경우, 링크로 업로드할 수 있는 파일의 최대 용량은 설정된 값으로 제한됩니다.
* 링크 업로드는 네트워크 장애 등으로 인해 업로드가 일시 중지된 상태에서 업로드를 재개하는 기능을 지원하지 않습니다.
*   링크의 상세 설정 화면에서 업로드를 사용함으로 설정한 경우, 업로드가 진행되는 동안 업로드 대상 경로가 삭제되면 업로드가 실패합니다. 또한 업로드 대상 폴더가 이동되어 이동된 위치의 상위 폴더 설정에 따라 접근 권한이 제거된 경우에도 업로드가 실패합니다.

    특히 대용량 파일 업로드로 인해 업로드에 시간이 많이 걸릴 것으로 예상되는 경우에는 주의가 필요합니다.
