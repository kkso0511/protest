# \[Mobile] 링크 수신 화면의 비밀번호 입력창에 링크 발신자의 이름을 표시하는 방법

### 개요 <a href="#a03" id="a03"></a>

이 매뉴얼에서는 iOS 애플리케이션 또는 Android 애플리케이션의 링크 수신 화면에 접근할 때 비밀번호 입력 화면에 발신자의 사용자명을 표시하는 방법에 대해 설명합니다.

***

### 제한·보충사항 <a href="#a04" id="a04"></a>

* 이 매뉴얼의 설정은 My Box의 파일을 선택하여 생성한 링크에도 적용됩니다.
* 비밀번호 입력 화면은 링크 생성 화면에서 비밀번호 항목에 체크한 상태로 전송된 링크에 접근하는 경우에만 표시됩니다.
* 링크 수신 메일에는 이 매뉴얼의 설정과 관계없이 발신자의 사용자명과 사용자 ID가 표시됩니다.
* 비밀번호 입력 화면에 사용자 ID를 표시할 수는 없습니다.
*   링크 접근 횟수는 세션 단위로 카운트되며, 그 횟수는 링크를 연 Web 브라우저에 Cookie로 저장됩니다.\
    따라서 한 사용자가 동일한 세션에서 링크를 10번 열어도 접근 횟수는 1회로 계산됩니다.

    브라우저의 방문 기록을 삭제하거나, 브라우저를 연 시점으로부터 24시간이 지나면 해당 세션은 종료됩니다.
* 전송한 링크를 열 때 두 번째 이후에도 비밀번호 입력을 요구하도록 설정할 수는 없습니다.
* 링크 생성 화면에서 표시되는 언어는 프로필의 설정 화면에서 변경할 수 있습니다.
* 링크 생성 이후 대상 폴더 또는 파일이 저장된 폴더에 대한 접근 권한이 삭제되면, 링크로 접속하더라도 해당 폴더나 파일에 접근할 수 없습니다.\
  또한 링크 생성자의 접근 권한이 삭제되면, 링크 수신자 역시 해당 폴더나 파일에 접근할 수 없습니다.
*   링크 수신 화면에서 파일을 미리보기 할 때 표시되는 날짜와 시간은 파일의 업데이트 시각이며, 이를 숨길 수는 없습니다.

    동일한 파일명으로 덮어쓰기 하면 링크 수신 화면에 표시되는 파일의 업데이트 시각도 변경됩니다.

***

### 절차 <a href="#a05" id="a05"></a>

{% hint style="warning" %}
**iOS 애플리케이션에서 설정하기**
{% endhint %}

1. [\[Mobile\] 링크 생성 화면을 표시하여 기본 기능을 확인하는 방법](https://help.directcloud.net/user_manual/undefined-4/mobile)에 따라 상세 설정 화면을 표시합니다.



2. '링크 생성자 정보 표시' 항목의 토글 스위치를 ON으로 설정합니다.\
   설정은 즉시 저장됩니다.



{% hint style="warning" %}
**Android 애플리케이션에서 설정**
{% endhint %}

1. [\[Mobile\] 링크 생성 화면을 표시하여 기본 기능을 확인하는 방법](https://help.directcloud.net/user_manual/undefined-4/mobile)에 따라 상세 설정 화면을 표시합니다.



2. '링크 생성자 정보 표시' 항목의 토글 스위치를 ON으로 설정합니다.\
   설정은 즉시 저장됩니다.



{% hint style="warning" %}
**비밀번호 입력 화면 확인**
{% endhint %}

수신자가 링크에 접근하면 비밀번호 입력 화면에 발신자의 사용자명이 표시됩니다.
