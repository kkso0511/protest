# 업로드 요청 폴더에 파일을 업로드하는 방법

### 개요 <a href="#a03" id="a03"></a>

이 매뉴얼에서는 메일로 수신하는 업로드 요청 URL을 클릭하여 업로드 요청 화면을 열고, 파일을 업로드하는 방법을 설명합니다.

***

### 동작 조건

* DirectCloud의 사용자 계정이 없어도 업로드 요청을 통해 파일을 업로드할 수 있습니다.
* 업로드 요청 화면은 어떤 기기에서 업로드 요청 URL을 클릭하더라도 항상 웹 애플리케이션에서 표시되며, 그 외의 애플리케이션에서는 표시되지 않습니다.

***

### 제한·보충사항 <a href="#a04" id="a04"></a>

* 업로드 요청 비밀번호 입력 화면에서는 Ctrl+V로 붙여넣기만 지원합니다.\
  마우스 오른쪽 클릭 등으로 표시되는 컨텍스트 메뉴의 붙여넣기는 지원하지 않습니다.
* 업로드 요청 링크 URL 전송 시 비밀번호 체크를 해제했다면, 비밀번호 입력 화면은 표시되지 않습니다.
* 업로드 요청 업로드 화면에서는 폴더를 업로드할 수 없습니다.
* 또한 DirectCloud-SHIELD IRM으로 암호화된 파일은 DirectCloud에 업로드할 수 없습니다.
* 업로드 요청 알림 메일의 발신자 정보에 표시되는 이름은 no-reply 메일에 회신할 수 있도록 설정되어 있는지 여부에 따라 달라집니다.
  * no-reply 메일에 회신할 수 있는 경우: 사용자 이름 no-reply@directcloud.jp
  * no-reply 메일에 회신할 수 없는 경우: DirectCloud no-reply@directcloud.co.jp
* 업로드 요청 업로드 화면에서 파일을 선택해 전송 버튼을 클릭하고, 마지막에 확인 버튼을 클릭한 시점에 파일이 업로드됩니다.
* 업로드 대상 폴더에 같은 이름의 파일이 이미 있는 경우, 같은 이름의 파일 업로드 동작 설정과 관계없이 파일명 끝에 숫자가 자동으로 추가되어 별도의 파일로 업로드됩니다.\
  DirectCloud AI가 활성화된 폴더나 문서 관리 메뉴 하위 폴더(DirectCloud 전자장부보존법 대응을 이용 가능한 경우)에 업로드할 때도 동일하게 적용됩니다.
* 업로드 요청으로 파일이 업로드되었을 때 발송되는 알림 메일은 발송하지 않도록 변경할 수 없습니다.
* 파일이 업로드되면 업로드 대상 폴더에 접근 권한이 있는 모든 사용자에게 ‘공유 폴더에 파일이 업로드되었습니다.’라는 제목의 알림 메일이 전송됩니다. 다만, 업로더 접근 권한이 부여된 사용자에게는 발송되지 않습니다.
* 모바일 기기에서는 웹 브라우저로 업로드 요청 업로드 화면을 열어 100MB 이상의 파일을 업로드할 수 없습니다.

***

### 절차 <a href="#a05" id="a05"></a>

{% hint style="warning" %}
**업로드 요청 화면 표시**
{% endhint %}

1. "\[DirectCloud] 업로드를 요청합니다."라는 제목의 이메일의 내용 중 '파일 업로드 하러가기'의 링크를 클릭합니다.

<figure><img src="../../.gitbook/assets/image (1111).png" alt=""><figcaption></figcaption></figure>

&#x20;      업로드 요청 URL에 비밀번호가 설정되어 있는 경우, 비밀번호 입력 화면이 표시됩니다.\
&#x20;      비밀번호 입력 화면이 표시되지 않는 경우에는 3단계로 진행하세요.



2. 비밀번호 송신 방법에 따라 비밀번호를 입력합니다.

❶ 비밀번호의 송신 방법으로서 '비밀번호 별도 이메일 발송'을 선택한 경우, 업로드 요청 폴더의 URL 안내 메일과는 별도로 "업로드를 위한 비밀번호가 도착했습니다."라고 하는 제목의 메일이 도착합니다.\
해당 이메일에 나열된 '비밀번호'를 입력하고 '확인' 버튼을 클릭합니다.

<figure><img src="../../.gitbook/assets/image (1112).png" alt=""><figcaption></figcaption></figure>

❷  업로드 요청 폴더의 비밀번호를 송신하는 방법으로 '비밀번호 동봉 발송' 을 선택한 경우\
업로드 요청 폴더의 URL 통지 메일에 기재되어 있는 비밀번호를 입력하고 '확인'버튼을 클릭합니다.



3. 업로드 요청 폴더 업로드 화면이 표시되는지 확인합니다.

<figure><img src="../../.gitbook/assets/image (1113).png" alt=""><figcaption></figcaption></figure>



{% hint style="warning" %}
**업로드 요청 폴더에 파일 업로드**
{% endhint %}

1. 업로드 요청 화면에서 '+추가' 버튼을 클릭하여 업로드할 파일을 선택하거나 회색 영역으로 파일을 드래그 앤 드롭합니다.

<figure><img src="../../.gitbook/assets/image (1114).png" alt=""><figcaption></figcaption></figure>

업로드할 파일이 설정됩니다.

<figure><img src="../../.gitbook/assets/image (1115).png" alt=""><figcaption></figcaption></figure>



2. 업로드 요청 URL의 발신자에게 메시지를 보내려면 메시지 텍스트 상자에 입력합니다.

<figure><img src="../../.gitbook/assets/image (1116).png" alt=""><figcaption></figcaption></figure>



3. '전송' 버튼을 클릭합니다.\
   업로드 프로세스의 상태 화면이 표시됩니다.



4. '업로드가 완료되었습니다.'라는 메시지가 표시되는지 확인하고 '확인' 버튼을 클릭합니다.\
   파일이 업로드됩니다.

<figure><img src="../../.gitbook/assets/image (1117).png" alt=""><figcaption></figcaption></figure>



**\[업로드된 파일 확인]**

DirectCloud 계정이 있는 사용자는 DirectCloud에 로그인하여 수신할 폴더를 표시하여 업로드된 파일을 확인할 수 있습니다.\
파일을 업로드한 사용자 입력란에 이메일 주소가 표시됩니다.

<figure><img src="../../.gitbook/assets/image (1118).png" alt=""><figcaption></figcaption></figure>



**\[알림 이메일 확인]**

업로드 요청 폴더에 파일을 업로드하면 파일을 업로드한 사용자와 수신 폴더에 업로드하도록 요청한 사용자 모두에게 알림 메일이 전송됩니다.

* 폴더에 파일을 업로드한 사용자에게 전송되는 알림 메일

<figure><img src="../../.gitbook/assets/image (1119).png" alt=""><figcaption></figcaption></figure>

* 폴더에 업로드를 요청한 사용자에게 전송되는 알림 이메일

![](<../../.gitbook/assets/스크린샷 2025-11-03 083844.png>)



{% hint style="warning" %}
**업로드 요청 화면의 표시 언어 변경**
{% endhint %}

1. 업로드 요청  화면의 오른쪽 상단에 표시되는 '한국어' 버튼을 클릭합니다.

<figure><img src="../../.gitbook/assets/image (1120).png" alt=""><figcaption></figcaption></figure>



2. 표시되는 언어 목록에서 변경할 언어를 선택합니다.

<figure><img src="../../.gitbook/assets/image (1121).png" alt=""><figcaption></figcaption></figure>



3. 회사 이름, 보낸 사람 이름, 파일 이름 이외의 표시가 선택한 언어로 변경되었는지 확인합니다.

<figure><img src="../../.gitbook/assets/image (1122).png" alt=""><figcaption></figcaption></figure>
