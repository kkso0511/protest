# \[Mobile] 링크 생성 화면에서 링크 URL을 수동으로 변경하는 방법

### 개요 <a href="#a03" id="a03"></a>

이 매뉴얼에서는 iOS 애플리케이션 또는 Android 애플리케이션의 링크 화면에서 생성된 링크 URL을 수동으로 변경하는 방법에 대해 설명합니다.

***

### 제한·보충사항 <a href="#a04" id="a04"></a>

* 과거에는 다른 폴더 파일에 대해 만든 링크 URL로 변경할 수 없습니다.  그러나 동일한 폴더 파일에 대해 게시된 링크인 경우 링크 기록에서 과거 URL로 변경할 수 있습니다. 자세한 내용은 [\[Mobile\] 링크 생성 화면을 표시하여 기본 기능을 확인하는 방법](https://help.directcloud.net/user_manual/undefined-4/mobile)을 참조하십시오.&#x20;
* 사용자 로그인 페이지의 기본 도메인을 설정하는 방법 의 단계에서 링크 URL의 도메인을 설정하면 생성된 링크 URL의 기본 도메인도 변경됩니다.
* 사용자 로그인 페이지의 기본 도메인을 설정하는 방법 에 따라 기본 도메인을 변경해도 변경 전의 공유 링크에 액세스할 수 있습니다.
* 링크에 대한 접근 횟수는 세션 단위로 집계되며, 그 횟수는 링크를 연 Web 브라우저에 Cookie로 저장됩니다.\
  따라서 한 사용자가 동일한 세션에서 링크에 10회 접근하더라도 접근 횟수는 1회로 계산됩니다.\
  브라우저의 방문 기록을 삭제하거나, 브라우저를 연 시점으로부터 24시간이 지나면 해당 세션은 종료됩니다.
* 전송한 링크를 열 때, 두 번째 이후에도 비밀번호 입력을 요구하도록 설정할 수는 없습니다.
* 링크 생성 후, 대상 폴더 또는 파일이 저장된 폴더에 대한 사용자 접근 권한이 제거되면 링크로 접속하더라도 해당 폴더나 파일에 접근할 수 없습니다.\
  또한 링크를 생성한 사용자의 접근 권한이 삭제되면, 링크를 받은 사용자 역시 해당 폴더나 파일에 접근할 수 없습니다.
* 링크 수신 화면에서 파일을 미리보기 할 때 표시되는 날짜와 시간은 파일의 업데이트 시각입니다. 이 날짜와 시간은 숨길 수 없습니다.\
  동일한 파일명으로 파일을 덮어쓰면, 링크 수신 화면에서도 해당 파일의 업데이트 시각이 변경됩니다.

***

### 절차

1. [\[Mobile\] 링크 생성 화면을 표시하여 기본 기능을 확인하는 방법](https://help.directcloud.net/user_manual/undefined-4/mobile)에 따라, URL을 변경하려는 폴더 또는 파일의 링크 화면을 엽니다.
2. 링크 URL에 표시된 연필 아이콘을 탭합니다.
3. 링크 URL 중 도메인명 뒤에 이어지는 문자열을 변경한 후 확인 버튼을 누릅니다.
