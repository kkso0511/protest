# DR 사이트에 대한 액세스를 허용 여부를  설정하는 방법

### 개요 <a href="#a03" id="a03"></a>

이 매뉴얼에서는 '재해 대책·원격지 백업' 옵션을 계약하고 있는 환경에서, 관리자 권한을 가지고 있지 않은 유저에 대해서, DR(Disaster Recovery) 사이트에 액세스의 허가 여부를 설정하는 방법 에 대해 설명합니다.

***

### 제한·보충사항 <a href="#a04" id="a04"></a>

* 관리자 권한이 부여된 사용자는 관리자 페이지에서 '스토리지' 항목을 사용할 수 있습니다.
* DR(Disaster Recovery) 서비스는 유료 옵션입니다. 사용을 희망하시는 경우 다이렉트클라우드 고객센터로 문의 주세요.
*   DirectCloud에서는 AWS 도쿄 리전 내의 아베일러빌리티 존(AZ: Availability Zone, 여러 데이터센터로 구성되며, 장애 발생 시 서로 영향을 받지 않도록 독립된 전원 설비와 네트워크 연결 기능을 갖춘 논리적 관리 단위) 3곳에 파일을 분산 저장하고 있습니다.\
    하지만 재해대책·원격지 백업에서는 이러한 분산된 데이터를 추가로 이중화하여 아래의 재해대책·원격지 백업 사이트를 구축합니다.\
    재해대책·원격지 백업을 계약할 때는 다음 중 하나의 리전을 선택할 수 있습니다.

    * 오사카 리전
    * 싱가포르 리전

    재해대책·원격지 백업의 사양은 DirectCloud 공통 사양 내의 재해대책·원격지 백업 사이트 관련 사양도 함께 참조해 주십시오.
* 재해대책·원격지 백업 사이트에서 가능한 작업은 파일 다운로드만입니다.\
  파일 업로드나 온라인 편집은 지원하지 않기 때문에 파일 버전 관리는 할 수 없습니다.
* DirectCloud의 재해대책·원격지 백업 사이트는 재해 등의 이유로 본番 사이트에 로그인할 수 없는 상황에서도 계속해서 파일을 사용할 수 있도록 하기 위한 목적이므로, 재해대책·원격지 백업 사이트에서 자동으로 파일을 복구하는 기능은 제공되지 않습니다.\
  재해대책·원격지 백업 사이트에서 다운로드하여 편집한 파일은 본 사이트 복구 후 수동으로 업로드해야 합니다.
* 재해대책·원격지 백업 사이트에서는 '공유설정' 메뉴 아래에 저장된 파일을 다운로드할 수 있습니다.
* Warm Storage를 사용할 수 있는경우, 재해대책·원격지 백업 사이트의 Warm Storage 메뉴에 저장된 파일을 다운로드할 수 있습니다.\
  다만 Warm Storage에서 파일을 다운로드하려면 유료 옵션을 계약해야 하며, Warm Storage의 다운로드 기능이 활성화되어 있어야 합니다.
* 재해 발생으로 DirectCloud의 본 사이트를 사용할 수 없게 된 경우에는, Warm Storage 다운로드 기능(유료 옵션)을 계약하지 않은 사용자도 재해대책·원격지 백업 사이트에서 Warm Storage 파일을 다운로드할 수 있게 됩니다.
* Cold Storage를 계약하고 있는 경우, 재해대책·원격지 백업 사이트에 Cold Storage 메뉴가 표시됩니다.\
  그러나 재해대책·원격지 백업 사이트에서는 Cold Storage의 파일을 다운로드할 수 없습니다.
* Warm Storage 또는 Cold Storage를 지정하여 재해대책·원격지 백업 서비스를 개별적으로 활성화하거나 비활성화할 수는 없습니다.
* 재해대책·원격지 백업 사이트에 적용되는 접근 제한은 IP 주소 제한과 이메일 기반의 이중 인증(2단계 인증)만 적용됩니다.  자세한 내용은 다음 항목을 참조해 주십시오.
  * [사용자 페이지에 로그인할 수 있는 IP 주소를 제한하는 방법](https://help.directcloud.net/admin_manual/images-and-media/ip)
  * [사용자가 2단계 인증을 사용하여 로그인 하도록 설정하는 방법](https://help.directcloud.net/admin_manual/images-and-media/2)
* 재해대책·원격지 백업 사이트에 접근할 수 있는 환경은 Web 브라우저만 해당됩니다.
* 재해대책·원격지 백업 사이트에 로그인하려면 본 사이트와 동일한 회사 ID, 사용자 ID, 비밀번호가 필요합니다.
* 사용자의 접근 허가 설정은 기본적으로 사용하지 않음으로 설정되어 있습니다.
* 다음에 해당하는 경우 재해대책·원격지 백업 사이트에 접근할 수 없습니다.
  * 오전 4:00\~6:00 사이(재해대책·원격지 백업 사이트와 본番 사이트의 데이터베이스 동기화가 실행되는 시간대)
  * 사전에 공지된 업데이트 및 유지 보수로 인한 서버 일시 중지
  * 복구 시간이 1시간 이내로 예상되는 일시적 장애
* 오사카 리전과 싱가포르 리전의 재해대책·원격지 백업 사이트와 도쿄 리전의 본番 사이트 간의 데이터베이스 동기화는 오전 4:00\~6:00 사이에 실행됩니다.\
  따라서 DirectCloud 관리자 페이지에서 설정한 내용이 즉시 재해대책·원격지 백업 사이트에 반영되는 것은 아닙니다.\
  위 시간대에 동기화가 완료된 이후에 재해대책·원격지 백업 사이트에서 설정 내용이 반영되었는지 확인해 주십시오.

***
