# 업로드 요청 이력에서 사용자가 발송한 업로드 요청 링크를 삭제하는 방법

### 개요 <a href="#a03" id="a03"></a>

이 매뉴얼에서는 '로그 현황'> '업로드 요청 내역' 메뉴에서 사용자가 보낸 업로드 요청 링크를 삭제하는 방법에 대해 설명합니다.

***

### 제한·보충사항 <a href="#a04" id="a04"></a>

* 관리자 권한이 부여된 사용자는 관리자 페이지에서 '로그 현황' 항목에 접근 가능합니다.
* 업로드 요청에는 폴더를 업로드할 수 없기 때문에, '업로드 요청' 화면에서 확인할 수 있는 것은 파일 업로드 이력만 해당됩니다.
* '로그 현황' > '업로드 요청 내역' 메뉴에는 Guest, LDAP 연동 및 SAML 연동으로 등록된 사용자의 업로드 요청 이력도 기록됩니다.
* DirectCloud의 옵션인 'DirectCloud-CONNECT'를 이용할 수 있는 경우라도, Connect 폴더에서는 공유 링크를 생성할 수 없기 때문에 Connect 폴더의 링크 설정 정보는 확인할 수 없습니다.
* 링크를 생성한 이력은 '파일의 조작' 로그에 기록됩니다.\
  자세한 내용은 [사용자의 파일 이용 내역을 확인하는 방법](https://help.directcloud.net/admin_manual/undefined-10/undefined-2)을 참조해 주십시오.
* 계약 담당 관리자 또는 일반 관리자에 의해 삭제된 링크는 복원할 수 없습니다.
* 계약 담당 관리자 및 일반 관리자에 대해 사용자가 생성한 링크를 삭제하지 못하도록 설정할 수는 없습니다.\
  단, 일반 관리자에 대해서는 관리자 기능 제한에서 '로그 현황'의 체크를 해제함으로써 '링크 현황 '메뉴에 접근할 수 없도록 설정할 수 있습니다.

***

### 절차 <a href="#a05" id="a05"></a>

1. '로그 현황' > '업로드 요청 내역'을 클릭합니다.

<figure><img src="../../.gitbook/assets/image (1518).png" alt=""><figcaption></figcaption></figure>



2. 삭제하려는 업로드 요청의 '동작' 열의 휴지통 버튼을 클릭합니다.

<figure><img src="../../.gitbook/assets/image (1520).png" alt=""><figcaption></figcaption></figure>



3. 화면에서 확인 버튼을 클릭합니다.

<figure><img src="../../.gitbook/assets/image (1522).png" alt=""><figcaption></figcaption></figure>



4. 해당업로드 요청의 '상태'가 '삭제'로 변경되었는지 확인합니다.

<figure><img src="../../.gitbook/assets/image (1523).png" alt=""><figcaption></figcaption></figure>
