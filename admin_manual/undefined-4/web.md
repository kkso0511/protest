# 접근 권한의 종류와 Web 브라우저에서 가능한 조작

### 개요 <a href="#a03" id="a03"></a>

DirectCloud에서는 부여된 접근 권한에 따라 사용자 또는 그룹이 폴더와 파일에 대해 수행할 수 있는 작업이 달라집니다.\
이 매뉴얼에서는 접근 권한 레벨의 종류와 Web 브라우저에서 가능한 작업에 대해 설명합니다.

***

### 제한·보충사항 <a href="#a04" id="a04"></a>

* 관리자 권한이 부여된 사용자는 관리자 페이지에서 '공유 설정' 메뉴를 사용할 수 있습니다.
* 1단계 폴더에 대한 작업은 Web 브라우저에서만 수행할 수 있습니다.
* Guest에게 설정할 수 있는 접근 권한 레벨은 사용자·그룹과 동일합니다.\
  하지만 Guest는 첨부 파일 전송, 업로드 요청, Guest 초대, Guest 초대 승인, 접근 권한 설정, 폴더 속성 설정은 사용할 수 없습니다.
* Guest는 파일 링크 생성, 폴더 링크 생성을 사용할 수 있지만, Guest에게 링크 생성 권한을 부여할지 여부는 [Guest의 링크 생성 가능 여부를 설정하는 방법](https://help.directcloud.net/admin_manual/undefined-3/guest-13)에서 제한할 수 있습니다.
* Guest는 1단계 폴더에 대한 작업을 할 수 없기 때문에 Guest에게 <마스터> 권한을 부여할 수 없습니다.
* Connect 폴더에서는 파일 링크 생성, 폴더 링크 생성, 첨부 파일 전송, Guest 초대를 사용할 수 없습니다.

***

### 절차 <a href="#a05" id="a05"></a>

공유자 권한의 종류와 권한별 제공 기능은 아래의 표를 참고해 주세요.

<figure><img src="../../.gitbook/assets/image (1938).png" alt=""><figcaption></figcaption></figure>

* 그룹권한과 사용자권한을 동시에 받아 복수권한인 경우 합집합으로 제공됩니다.
* Guest 사용자는 전체권한이라도 'Guest 초대' 기능을 사용할 수 없습니다.
* Guest 사용자는 전체권한, 편집자, 편집자- 라도 '업로드 요청', '파일첨부 메일발송' 기능을 사용할 수 없습니다.
* Guest 사용자는 관리자의 기능 제한 설정에 따라'링크' 기능을 사용할 수 없습니다.
* '편집자'의 파일삭제 권한은 본인이 업로드한 파일로 제한됩니다.
* '마스터'는 일부 권한이 제한될 수 있으며, 각 마스터별 관리범위는 별도로 확인할 수 있습니다.
* ※ 폴더별 설정에 따라'권한 관리'를 전체권한에게 부여할 수 있습니다. 단, Guest는 전체권한이라도 해당 기능을 사용할 수 없습니다.
