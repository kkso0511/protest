# 사용자의 액세스 권한과 액세스 수준을 일괄 적으로 추가하는 방법 (한국미제공)

### 개요 <a href="#a03" id="a03"></a>

템플리트에 따라 액세스 권한 추가를 위한 CSV 파일을 작성 및 임포트하여 일괄적으로 액세스 권한 및 액세스 레벨을 추가할 수 있습니다.\
이 섹션에서는 공유 및 연결 메뉴의 액세스 권한 일괄 처리 화면에서 공유 폴더, 연결 폴더에 대한 사용자의 액세스 권한 및 액세스 수준을 일괄 적으로 추가하는 방법에 대해 설명합니다.

### 제한·보충사항 <a href="#a04" id="a04"></a>

* 이 기사에서는 환경 설정 메뉴의 공유 및 연결 관리 메뉴를 설정하는 방법에 대해 설명합니다. 수 있습니다.
* 관리자 권한이 부여된 사용자는 [관리자의 기능 제한](https://boxmanager.directcloud.jp/setting/manager) 으로 "환경 설정"을 선택해야 합니다.
* Connect User는 액세스 권한과 액세스 레벨을 일괄 적으로 부여 할 수 없습니다.
* CSV 파일의 폴더 경로에 대상 사용자가 "지정된 사용자"와 "모든 사용자"의 폴더가 혼합되어 있으면 "지정된 사용자"의 폴더에 대한 액세스 권한 정보만 가져옵니다.\
  대상 사용자가 "모든 사용자"의 폴더를 처리하면 "존재하지 않는 폴더입니다."라는 오류가 표시됩니다.
* 이미 동일한 사용자 게스트 그룹에 액세스 권한이 등록되어 있으면 "등록된 액세스 권한입니다."라는 오류가 표시됩니다.\
  액세스 권한 수정에 대한 자세한 내용은 [사용자의 액세스 권한 및 액세스 레벨을 일괄적으로 변경하는 방법](https://help.directcloud.jp/s/article/operation-of-administrator-function11145) 을 참조하십시오.
* CSV 가져오기에서 "액세스 권한 추가"를 선택하고 아래에 해당하는 CSV 파일을 가져오면 해당 행 가져오기가 실패하고 "필수 항목이 올바르게 입력되지 않았습니다."라는 오류 메시지가 표시됩니다. 합니다.
  * 제1계층\~제6계층에 존재하지 않는 폴더명이 입력되어 있는 경우
  * 입력한 폴더의 상위 계층에 있는 폴더의 이름을 입력하지 않은 경우
  * 사용자 ID, 게스트 ID, 그루브 이름이 입력되지 않은 경우
  * 존재하지 않는 권한 이름이 입력된 경우
  * 권한이 입력되지 않은 경우
  * 첫 번째 계층의 폴더 권한에 "noAccess"가 입력된 경우
* CSV 가져오기에서 "액세스 권한 추가"를 선택하고 아래에 해당하는 CSV 파일을 가져오면 해당 행 가져오기가 실패하고 "등록되지 않은 사용자 ID입니다."라는 오류 메시지가 표시됩니다. .
  * 존재하지 않는 사용자 ID, 게스트 ID가 입력된 경우
* CSV 가져오기에서 "액세스 권한 추가"를 선택하고 아래에 해당하는 CSV 파일을 가져오면 해당 행 가져오기가 실패하고 "존재하지 않는 그룹입니다."라는 오류 메시지가 표시됩니다.
  * 존재하지 않는 그룹 이름을 입력한 경우
* 일괄 처리 권한 추가로 사용자 액세스 권한이 개별적으로 설정된 폴더에 대해 "상위 계층의 권한 상속을 안 함" 설정을 한 CSV 파일을 가져오면 해당 행 가져오기가 실패합니다. , "상위 계층에 액세스 권한이 설정되어 있지 않습니다."라는 오류 메시지가 표시됩니다.\
  이 경우 "CSV 가져오기"에서 다음 순서로 설정하십시오.
  * "액세스 권한 삭제"를 선택하고 액세스 권한을 포함하여 필요한 정보를 입력한 CSV 파일을 가져와 해당 폴더의 액세스 권한을 삭제합니다.
  * "액세스 권한 추가"를 선택하고 "상위 계층의 권한 상속 없음" 설정을 사용하여 CSV 파일을 가져옵니다.
* 일괄 처리에서 상위 계층 폴더에서 상속된 그룹의 액세스 권한에서 특정 사용자나 그룹을 제외하고 싶은 경우 [사용자의 액세스 권한 및 액세스 수준을 일괄 삭제하는 방법](https://help.directcloud.jp/s/article/operation-of-administrator-function11146) 으로 액세스 권한을 일괄 삭제할 수 있습니다. 이 절차에서 액세스 권한 일괄 추가용 CSV 파일의 "권한"에 "noAccess"를 설정하여 액세스 권한을 제외할 수 있습니다.
* 액세스 권한 CSV 가져오기의 행 수 한도는 다음과 같이 계약 계획에 따라 다릅니다.

| 고급   | 사업     | 프리미엄   | 엔터프라이즈 |
| ---- | ------ | ------ | ------ |
| 500줄 | 1,000줄 | 3,000줄 | 5,000줄 |

* 가져오기를 처리하는 동안 추가로 CSV 가져오기를 요청할 수 없습니다.
* 문자 코드가 "Shift\_JIS"인 CSV 파일 또는 텍스트 파일에 X 0213:2004(통칭 JIS2004) 표준으로 변경된 구체의 문자를 입력하면 저장할 때 문자가 깨집니다. 그 상태에서 CSV 또는 텍스트 파일을 가져오면 깨진 상태로 가져옵니다.\
  깨짐 방지를 위해 문자 코드에 UTF-8 BOM 포함을 사용하는 것이 좋습니다.
* 내보낼 액세스 권한 정보 및 템플릿의 문자 코드는 환경 설정 > 고급 설정 메뉴의 내보내기 시 문자 코드 설정에서 설정됩니다.\
  자세한 내용은 [내보낼 파일의 문자 코드를 설정하는 방법을](https://help.directcloud.jp/s/article/operation-of-administrator-function11267) 참조하십시오.
* 계약 플랜이 엔터프라이즈이고 DirectCloud의 무료 옵션인 DirectCloud-CONNECT를 사용할 수 있는 경우 관리 페이지의 환경설정 > Connect 관리 메뉴에서 액세스 권한을 일괄 추가할 수 있습니다.
* 계약 계획이 비즈니스 이상이고 DirectCloud 유료 옵션인 DirectCloud-SHIELD DLP를 계약하는 경우 관리 페이지의 환경 설정 > DLP 관리 메뉴에서 액세스 권한을 일괄 추가할 수 있습니다. 수 있습니다.
* 계약 플랜이 비즈니스 이상이고 DirectCloud의 무료 옵션인 "DirectCloud 전자장부 보존법 대응"을 이용할 수 있는 경우, 관리 페이지의 "환경 설정" > "문서 관리" 메뉴에서 액세스권을 일괄로 추가하는 것이 수 있습니다.
* 문서 관리 폴더에서는 게스트를 초대할 수 없으므로 문서 관리 메뉴의 액세스 권한 일괄 처리에서 사용하는 CSV 파일에 게스트 ID 항목이 포함되지 않습니다.
* 계약 계획이 비즈니스 이상이고 DirectCloud의 유료 옵션인 "웜 스토리지"를 계약하는 경우 관리 페이지의 "환경 설정" > "웜 스토리지 관리" 메뉴에서 액세스 권한을 일괄 추가할 수 있습니다. 합니다.
* 웜 스토리지 폴더에서는 게스트를 초대할 수 없으므로 웜 스토리지 메뉴의 액세스 권한 일괄 처리에서 사용하는 CSV 파일에 게스트 ID 항목이 포함되지 않습니다.

#### 공유 폴더에 사용자의 액세스 권한 및 액세스 수준을 일괄 추가

**"액세스 권한 일괄 처리"화면 표시**

1. 환경설정 > 공유 메뉴를 선택하고 액세스 권한 일괄 처리 버튼을 클릭합니다.

![영상](https://help.directcloud.jp/servlet/rtaImage?eid=ka0Q80000001hcP\&feoid=00N2w00000JMb2T\&refid=0EMQ8000005g55F)

**액세스 권한 일괄 처리 템플릿 다운로드**

1. 액세스 권한 일괄 처리 화면의 CSV 가져오기 설정에서 템플릿 버튼을 클릭합니다.

![영상](https://help.directcloud.jp/servlet/rtaImage?eid=ka0Q80000001hcP\&feoid=00N2w00000JMb2T\&refid=0EMBB000000OQjq)

1. 다운로드한 템플릿을 편집하여 액세스 권한 추가를 위한 CSV 파일을 만듭니다. CSV 파일에는 액세스 권한을 추가하려는 사용자, 게스트 및 그룹 정보만 포함해야 합니다.

**액세스 권한 추가를 위한 CSV 파일 가져오기**

1. \[액세스 권한 일괄 처리] 화면의 \[CSV 가져오기] 설정에서 \[액세스 권한 추가]를 선택하고 \[파일 선택] 버튼을 클릭하여 만든 액세스 권한 추가를 위한 CSV 파일을 선택합니다.

![영상](https://help.directcloud.jp/servlet/rtaImage?eid=ka0Q80000001hcP\&feoid=00N2w00000JMb2T\&refid=0EMBB000000OQlS)

2. 업로드 버튼을 클릭합니다.

![영상](https://help.directcloud.jp/servlet/rtaImage?eid=ka0Q80000001hcP\&feoid=00N2w00000JMb2T\&refid=0EMBB000000OQlh)

3. CSV 가져오기 확인 화면에서 파일 이름, 등록 행 수, 처리 유형을 확인한 후 지금 요청 버튼을 클릭합니다.\
   나중에 요청 버튼을 클릭하면 액세스 권한 추가 [최종 요청을 제출하는](https://help.directcloud.jp/s/article/operation-of-administrator-function11144#b01) 단계에서 최종 요청을 제출하고 액세스 권한을 가져옵니다.

![영상](https://help.directcloud.jp/servlet/rtaImage?eid=ka0Q80000001hcP\&feoid=00N2w00000JMb2T\&refid=0EMBB000000OQm1)

CSV 파일에 나열된 액세스 권한을 가져옵니다.\
액세스 권한 추가가 완료되면 "액세스 권한 CSV 가져오기가 완료되었습니다."라는 화면이 표시됩니다.

4. 오류가 없는지 확인하고 확인 버튼을 클릭합니다.

![영상](https://help.directcloud.jp/servlet/rtaImage?eid=ka0Q80000001hcP\&feoid=00N2w00000JMb2T\&refid=0EMBB000000OQjS)

**액세스 권한 추가 최종 요청 제출**

액세스 권한 추가를 위한 CSV 파일 가져오기에서 '나중에 요청'을 선택한 경우 가져오기 요청을 만든 계약 담당 관리자 또는 일반 관리자의 이메일 주소로 최종 요청 링크가 포함됩니다. 알림 메일이 도착합니다.\
다음 단계에 따라 최종 요청을 제출하고 액세스 권한을 추가합니다.

1. 알림 이메일을 보고 최종 요청 URL을 클릭합니다.

![영상](https://help.directcloud.jp/servlet/rtaImage?eid=ka0Q80000001hcP\&feoid=00N2w00000JMb2T\&refid=0EMBB000000OQnY)

CSV 가져오기 요청 화면이 표시됩니다.

2. 요청 버튼을 클릭합니다.

![영상](https://help.directcloud.jp/servlet/rtaImage?eid=ka0Q80000001hcP\&feoid=00N2w00000JMb2T\&refid=0EMBB000000OQns)

CSV 파일을 가져오고 나열된 액세스 권한을 추가합니다.\
액세스 권한 추가가 완료되면 "액세스 권한 CSV 가져오기가 완료되었습니다."라는 화면이 표시됩니다.

3. 오류가 없는지 확인하고 확인 버튼을 클릭합니다.

![영상](https://help.directcloud.jp/servlet/rtaImage?eid=ka0Q80000001hcP\&feoid=00N2w00000JMb2T\&refid=0EMBB000000OQoR)

#### Connect 폴더에 사용자의 액세스 권한 및 액세스 수준을 일괄 추가

**"액세스 권한 일괄 처리"화면 표시**

1. 환경설정 > Connect 관리 메뉴를 선택하고 액세스 권한 일괄 처리 버튼을 클릭합니다.

![영상](https://help.directcloud.jp/servlet/rtaImage?eid=ka0Q80000001hcP\&feoid=00N2w00000JMb2T\&refid=0EMQ8000005g5mn)

액세스 권한 일괄 처리 화면이 표시됩니다.

**액세스 권한 일괄 처리 템플릿 다운로드**

1. 액세스 권한 일괄 처리 화면의 CSV 가져오기 설정에서 템플릿 버튼을 클릭합니다.

**액세스 권한 추가를 위한 CSV 파일 가져오기**

1. \[액세스 권한 일괄 처리] 화면의 \[CSV 가져오기] 설정에서 \[액세스 권한 추가]를 선택하고 \[파일 선택] 버튼을 클릭하여 만든 액세스 권한 추가를 위한 CSV 파일을 선택합니다.

![영상](https://help.directcloud.jp/servlet/rtaImage?eid=ka0Q80000001hcP\&feoid=00N2w00000JMb2T\&refid=0EMBB000000OQvh)

2. 업로드 버튼을 클릭합니다.

![영상](https://help.directcloud.jp/servlet/rtaImage?eid=ka0Q80000001hcP\&feoid=00N2w00000JMb2T\&refid=0EMBB000000OQwV)

3. CSV 가져오기 확인 화면에서 파일 이름, 등록 행 수, 처리 유형을 확인한 후 지금 요청 버튼을 클릭합니다.\
   나중에 요청 버튼을 클릭하면 액세스 권한 추가 [최종 요청을](https://help.directcloud.jp/s/article/operation-of-administrator-function11144#b02) 제출하는 단계에서 최종 요청을 제출하고 액세스 권한을 가져옵니다.

![영상](https://help.directcloud.jp/servlet/rtaImage?eid=ka0Q80000001hcP\&feoid=00N2w00000JMb2T\&refid=0EMBB000000OQvL)

CSV 파일에 나열된 액세스 권한을 가져옵니다.\
액세스 권한 추가가 완료되면 "액세스 권한 CSV 가져오기가 완료되었습니다."라는 화면이 표시됩니다.

4. 오류가 없는지 확인하고 확인 버튼을 클릭합니다.

![영상](https://help.directcloud.jp/servlet/rtaImage?eid=ka0Q80000001hcP\&feoid=00N2w00000JMb2T\&refid=0EMBB000000OQwp)

**액세스 권한 추가 최종 요청 제출**

액세스 권한 추가를 위한 CSV 파일 가져오기에서 '나중에 요청'을 선택한 경우 가져오기 요청을 만든 계약 담당 관리자 또는 일반 관리자의 이메일 주소로 최종 요청 링크가 포함됩니다. 알림 메일이 도착합니다.\
다음 단계에 따라 최종 요청을 제출하고 액세스 권한을 추가합니다.

1. 알림 이메일을 보고 최종 요청 URL을 클릭합니다.

![영상](https://help.directcloud.jp/servlet/rtaImage?eid=ka0Q80000001hcP\&feoid=00N2w00000JMb2T\&refid=0EMBB000000OQx4)

CSV 가져오기 요청 화면이 표시됩니다.

2. 요청 버튼을 클릭합니다.

![영상](https://help.directcloud.jp/servlet/rtaImage?eid=ka0Q80000001hcP\&feoid=00N2w00000JMb2T\&refid=0EMBB000000OQx9)

CSV 파일을 가져오고 나열된 액세스 권한을 추가합니다.\
액세스 권한 추가가 완료되면 "액세스 권한 CSV 가져오기가 완료되었습니다."라는 화면이 표시됩니다.

3. 오류가 없는지 확인하고 확인 버튼을 클릭합니다.

![영상](https://help.directcloud.jp/servlet/rtaImage?eid=ka0Q80000001hcP\&feoid=00N2w00000JMb2T\&refid=0EMBB000000OQxO)
