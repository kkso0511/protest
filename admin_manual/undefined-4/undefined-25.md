# 사용자 페이지에서도 폴더에 대한 접근 권한을 설정할 수 있도록 하는 방법

### 개요 <a href="#a03" id="a03"></a>

이 매뉴얼에서는 사용자가 폴더의 접근 권한과 접근 권한 레벨을 설정할 수 있도록, 특정 폴더를 지정해 접근 권한 설정을 확장하는 방법에 대해 설명합니다.

***

### 제한·보충사항 <a href="#a04" id="a04"></a>

* 관리자 권한이 부여된 사용자는 관리자 페이지의 '공유 설정'을 사용할 수 있습니다.
* 초기 상태에서는 접근 권한 설정 확장이 사용하지 않음으로 설정되어 있습니다.
* Warm Storage를 이용할 수 있다면, 관리 페이지에 Warm Storage 관리 메뉴가 표시됩니다.
* Cold Storage를 계약한 경우, 관리 페이지에 Cold Storage 관리 메뉴가 표시됩니다.
* 접근 권한 설정 확장은 최상위 계층의 폴더에 대해 설정하며, 설정한 내용은 하위 폴더에 그대로 적용됩니다.
* Web 브라우저에서 사용자에게 접근 권한을 부여할 수 있도록 하려면, 아래와 같이 설정되어 있어야 합니다.
  * 이 매뉴얼의 절차에서 접근 권한 설정 확장을 사용함으로 설정
  * 사용자에게 전체권한 접근 권한 레벨을 부여
* 사용자가 접근 권한을 설정할 수 있는 기능은 Web 브라우저에서만 제공합니다.
* 이 매뉴얼의 설정으로 접근 권한 설정을 확장하면, Web 브라우저의 아래 화면에서 각각 최대 3,000건까지 접근 권한을 설정할 수 있게 됩니다.
  * 사용자 접근 권한 설정 화면
  * Guest 접근 권한 설정 화면
* Cold Storage 폴더에서 설정할 수 있는 접근 권한의 수는 최대 100명까지이며, 단 마스터는 제외됩니다.
* Guest에게는 접근 권한 설정 확장을 적용할 수 없습니다.
* 사용자가 설정한 접근 권한은 '로그 현황' > '전체권한자 권한변경 내역' 메뉴에서 확인할 수 있습니다.
* '전체권한자 권한변경 내역' 메뉴에서는 누가 어떤 폴더의 권한을 변경했는지는 확인할 수 있지만, 누구에게 어떤 권한을 설정했는지는 확인할 수 없습니다.

***

### 절차 <a href="#a05" id="a05"></a>

{% hint style="info" %}
**Shared Box 폴더를 지정하여 권한관리 확대 설정**
{% endhint %}

1. '공유 설정' > 'Shared Box 관리' 관리 메뉴를 선택하고 권한 설정을 확장할 폴더를 클릭합니다.

<figure><img src="../../.gitbook/assets/image (1969).png" alt=""><figcaption></figcaption></figure>



2. 화면을 아래로 스크롤하고 폴더 속성 설정에서 '권한관리 확대'의 '수정' 버튼을 클릭합니다.

<figure><img src="../../.gitbook/assets/image (1970).png" alt=""><figcaption></figcaption></figure>



3. 권한관리 확대에서 사용을 선택하고 저장 버튼을 클릭합니다.

<figure><img src="../../.gitbook/assets/image (1971).png" alt=""><figcaption></figcaption></figure>

사용자 페이지에서  전체권한이 부여된 폴더 속성에 '그룹/사용자 권한관리' 및 'Guest 권한관리'가 표시됩니다.
