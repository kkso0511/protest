# DirectCloud AI 사양 및 참고 사항

### 개요 <a href="#a03" id="a03"></a>

DirectCloud에서는 생성형 AI를 활용하여 Storage에서 필요한 정보를 빠르게 찾아내거나 요약할 수 있는 옵션 서비스인 DirectCloud AI를 제공합니다.\
web 브라우저의 DirectCloud AI가 활성화된 폴더에서 채팅을 시작하고 알고 싶은 내용을 질문하면, 생성형 AI가 해당 폴더 내 여러 파일을 참조하여 정보를 추출하고 제시하는 방식으로 작동합니다.\
또한 DirectCloud AI에는 이메일을 파일로 저장할 때 사용하는 파일 형식인 EML 파일을, EML 요약 기능이 활성화된 폴더에 업로드하면 그 내용을 질문과 답변 형식으로 요약하여 텍스트 파일로 저장하는 기능이 있습니다. 요약된 텍스트 파일 또한 DirectCloud AI의 채팅에서 참조됩니다.\
이 매뉴얼에서는 DirectCloud가 제공하는 웹 매뉴얼에 기재된 DirectCloud AI 관련 용어의 정의와 함께, DirectCloud AI의 사양 및 주의 사항 등에 대해 설명합니다.

### 제한·보충사항 <a href="#a04" id="a04"></a>

* Microsoft Azure 서버의 부하가 높아질 경우, 벡터 데이터 생성이나 채팅 응답이 실패할 수 있습니다.

{% hint style="info" %}
**DirectCloud AI 관련 용어**
{% endhint %}

* **DirectCloud AI**\
  DirectCloud에서 제공하는 문서 생성에 특화된 AI 서비스입니다. ‘문서 생성 AI’라고도 불립니다.\
  학습 데이터와 DirectCloud에 저장된 문서 파일을 기반으로 생성형 AI가 정확도가 높은 답변을 생성합니다. 이 서비스는 Azure OpenAI Service의 API를 이용해 구현되어 있습니다.\
  DirectCloud AI는 처리 대상 파일의 데이터 청크를 생성하고, 이를 생성형 AI가 이해할 수 있는 형태로 벡터화하여 대화방에서 사용자의 질문에 대한 답변을 생성합니다.
* **생성형 AI**\
  대규모 언어 모델(LLM, Large Language Model) 기술을 이용해 대량의 텍스트나 이미지 등의 콘텐츠를 생성하는 대화형 AI 서비스입니다.
* **Azure OpenAI Service**\
  OpenAI의 ChatGPT에도 사용되고 있는 AI 모델을 Microsoft Azure 환경에서 사용할 수 있도록 Microsoft가 제공하는 서비스입니다.\
  API 형태로 제공되며, 사용을 위해서는 API 키와 API URI(엔드포인트 URL)가 필요합니다. 또한 Azure AI Foundry에서 임베딩 모델과 GPT 모델 정보를 설정해야 합니다.\
  OpenAI 및 Azure OpenAI Service가 제공하는 모든 AI 모델을 DirectCloud AI에서 사용할 수 있는 것은 아닙니다.
* **임베딩 모델**\
  텍스트 데이터를 벡터 데이터로 변환하는 데 사용되는 모델입니다.
* **GPT 모델**\
  OpenAI가 개발한 대규모 언어 모델로, 학습 데이터나 추출된 데이터 청크를 기반으로 자연스러운 답변을 생성합니다.
* **벡터 데이터**\
  텍스트 데이터를 생성형 AI가 이해할 수 있는 수치 배열로 변환한 데이터입니다.\
  또한 텍스트 데이터를 벡터 데이터로 변환하는 과정을 ‘벡터화’라고 부릅니다.
* **데이터 청크**\
  DirectCloud AI에서 처리 대상 파일의 텍스트 데이터를 작은 단위로 분할한 것을 의미합니다.\
  큰 데이터를 여러 데이터 청크로 분할하고, 이를 벡터화함으로써 효율적으로 정보를 검색하고 정확한 정보를 추출할 수 있습니다.\
  DirectCloud AI에서는 데이터 청크를 편집함으로써 보다 적절한 답변을 반환할 수 있습니다.
* **토큰**\
  DirectCloud AI가 텍스트를 처리할 때 사용하는 최소 단위입니다.\
  히라가나는 한 글자당 1토큰, 한자는 한 글자당 2\~3토큰, 영어는 한 단어당 1토큰이 소모됩니다.
* **크레딧**\
  AI 서비스(DirectCloud의 경우 Azure OpenAI Service) 이용에 설정된 제한값을 의미합니다.\
  할당된 크레딧 잔여량 및 유효 기간을 초과하면 AI 서비스를 이용할 수 없습니다.



{% hint style="info" %}
**관리 페이지의 DirectCloud AI 설정 사양**
{% endhint %}

**DirectCloud AI 설정에 관한 사양**

* DirectCloud AI 설정은 기본적으로 ‘사용하지 않음’으로 설정되어 있습니다.\
  관리 페이지의 지정된 폴더에서 DirectCloud AI를 활성화하거나, 사용자 페이지의 지정된 폴더에서 DirectCloud AI를 활성화하여 ‘사용함’으로 설정된 폴더가 있는 경우, 관리 페이지의 'DirectCloud AI' > '설정' 메뉴에서는 ‘사용하지 않음’을 선택할 수 없습니다.
* 'DirectCloud AI' > '설정' 메뉴에서 선택한 GPT 모델과 다른 GPT 모델의 정보를 배포 이름(Deploy Name)에 입력하면, ‘잘못된 API 정보가 입력되었습니다.’라는 오류 메시지가 표시됩니다.\
  이 경우 Azure AI Foundry를 확인하여 설정된 배포 이름을 정확히 입력해야 합니다.<br>

**추천 GPT 모델에 대해서**

* 선택하는 GPT 모델에 따라 성능과 이용 요금이 크게 달라질 수 있습니다.
* DirectCloud AI에서는 성능과 운영 비용을 고려하여 다음의 GPT 모델을 추천하고 있습니다.\
  \
  GPT 모델 설정(질문·응답 기능):\
  GPT-4o\
  \
  GPT 모델 설정(EML 파일 요약 기능):\
  GPT-4o mini\
  \
  단, 2025년 9월 26일 현재, GPT-4o 이외의 모델에서는 일본 국내 리전을 지정할 수 없습니다.\
  Azure AI Foundry에서 모델을 배포할 때 ‘배포 유형’에서 ‘글로벌 표준’을 선택하면 GPT-4o mini, o3 mini, GPT-4.1, GPT-4.1-mini를 사용할 수 있습니다.\
  그러나 이 경우, 글로벌 표준에 대응하는 해외 데이터 센터에서 데이터 처리가 이루어질 가능성이 있으므로 주의하시기 바랍니다.\
  \
  자세한 내용은 Azure OpenAI Service의 문서를 참조하시기 바랍니다.\
  또한, EML 파일 요약 기능만 비활성화하도록 설정할 수는 없습니다.<br>

**답변의 평가에 대해서**

* 기본적으로 ‘답변 평가’ 기능은 모든 DirectCloud AI 플랜에서 비활성화되어 있습니다.\
  관리 페이지의 DirectCloud AI > 설정 메뉴에서 ‘답변 평가’ 기능을 활성화하면, 관리 페이지와 사용자 페이지의 폴더 속성에서는 해당 기능을 비활성화로 변경할 수 없습니다.&#x20;



{% hint style="info" %}
**폴더에 대한 DirectCloud AI 설정 사양**
{% endhint %}

**폴더 속성 설정에 대해**

* 폴더의 DirectCloud AI 설정은 기본적으로 ‘사용하지 않음’으로 설정되어 있습니다.
* 폴더의 DirectCloud AI 설정 정보는 하위 폴더에 상속되지 않으며, 폴더마다 개별적으로 설정해야 합니다.
* DirectCloud AI 설정은 1단계부터 6단계까지의 폴더에서 설정할 수 있습니다.
* 폴더 속성의 ‘용량 제한’이 DirectCloud AI의 ‘파일 총용량’보다 작은 값으로 설정되어 있는 경우, ‘용량 제한’ 설정이 적용됩니다.
* Shared Box 이외의 폴더에서는 DirectCloud AI를 활성화할 수 없습니다.
* 사용자 페이지의 폴더 속성에서도 DirectCloud AI를 활성화할 수 있습니다.
* DirectCloud AI가 ‘사용함’으로 설정된 폴더에서는 온라인 편집 및 버전 관리 기능을 이용할 수 없습니다.
* DirectCloud AI 설정을 ‘사용하지 않음’으로 변경한 뒤 다시 ‘사용함’으로 되돌리더라도, 폴더 속성의 ‘버전 관리’ 설정은 유지됩니다.
* 다음의 설정이나 작업을 수행하면 DirectCloud AI를 설정할 수 있는 폴더 수가 증가합니다.\
  · DirectCloud AI 설정을 ‘사용하지 않음’으로 변경\
  · DirectCloud AI 설정이 활성화된 폴더를 Shared Box 메뉴 이외의 폴더로 이동

**AI 요약에 대해**

* 폴더의 DirectCloud AI 설정 중 ‘AI 요약’은 기본적으로 사용하지 않음으로 설정되어 있습니다.
* 폴더의 DirectCloud AI 설정에서 ‘AI 요약’을 선택하면, 해당 폴더에 확장자가 eml인 파일을 업로드하거나 복사, 이동할 때 그 내용이 질문과 답변 형식으로 요약되어 텍스트 파일로 저장됩니다. 원래의 EML 파일은 ‘EML’ 폴더로 이동됩니다.
* 요약된 텍스트 파일 역시 벡터 데이터 생성 대상에 포함됩니다.\
  또한 ‘AI 요약’을 선택하더라도, 기존 폴더에 이미 벡터 데이터가 생성된 EML 파일은 요약되지 않습니다. 해당 폴더에 EML 파일을 업로드, 복사, 이동할 때 요약이 시작됩니다.



{% hint style="info" %}
**파일 업로드 및 벡터 데이터에 관한 사양**
{% endhint %}

이 매뉴얼에서는 DirectCloud AI가 활성화된 폴더에 파일을 업로드, 이동, 복사할 때의 주의사항과, DirectCloud AI가 처리 대상으로 하는 벡터 데이터에 관한 유의사항을 설명합니다.

**파일 업로드, 이동, 복사에 대해**

* DirectCloud AI가 활성화된 폴더에는 확장자가 pdf, docx, txt, eml인 파일을 업로드할 수 있습니다.\
  · 확장자가 pdf, docx, txt, eml이 아닌 파일은 업로드가 완료되지 않으며 오류가 발생합니다.\
  · PDF, docx(Microsoft Word 문서), 텍스트 형식, EML 형식이 아닌 파일을 pdf, docx, txt, eml 확장자로 업로드한 경우 업로드는 완료되지만, 벡터 데이터는 생성되지 않습니다. 이 경우 파일에는 처리 실패를 의미하는 느낌표( ! ) 아이콘이 표시됩니다.
* DirectCloud AI가 활성화된 폴더에 동일한 이름의 파일을 업로드해도 기존 파일은 덮어쓰여지지 않습니다. 동일한 이름의 파일을 업로드했을 때의 처리 설정과 관계없이 파일 이름 끝에 ‘-복사본’이 자동으로 추가되어 별도의 파일로 업로드됩니다.
* AI 요약이 활성화된 폴더에 동일한 이름의 EML 파일을 업로드한 경우에도 덮어쓰여지지 않으며, 생성된 텍스트 파일 이름 끝에 ‘-복사본’이 추가되어 “<파일명> - 복사본.eml.txt”라는 이름으로 저장됩니다. 또한 원본 EML 파일은 “<파일명> - 복사본.eml”이라는 이름으로 EML 폴더로 자동 이동됩니다.
* DirectCloud 드라이브를 이용해 DirectCloud AI가 활성화된 폴더에 파일을 업로드하는 경우, 최신 버전의 애플리케이션을 설치할 것을 권장합니다.\
  이전 버전의 애플리케이션에서는 업로드 제한이 정상적으로 작동하지 않을 수 있습니다.

**벡터 데이터에 관한 사양에 대해**

* DirectCloud AI의 생성형 AI는 벡터 데이터를 생성하고 단어 등을 수치화하여 언어를 처리합니다.\
  생성된 벡터 데이터는 파일 형태로 저장되지 않기 때문에 사람이 직접 확인할 수 없습니다.
* 벡터 데이터는 DirectCloud AI가 활성화된 폴더에 파일을 업로드, 복사, 이동할 때 자동으로 생성됩니다.\
  파일 내 텍스트 정보의 단어 수나 문장 구조, 분량에 따라 많은 토큰이나 크레딧이 소비될 수 있으므로 주의가 필요합니다.
* DirectCloud에서 파일이 잠금 상태인 경우, 해당 파일을 DirectCloud AI가 활성화된 폴더로 이동하거나 복사하더라도 벡터 데이터가 생성되므로 토큰 또는 크레딧의 소비에 유의해야 합니다.
* DirectCloud AI 설정 변경 시 벡터 데이터의 상태는 다음과 같습니다.
  * 폴더의 DirectCloud AI 설정을 ‘사용함’으로 변경하면, 폴더 내에 저장된 파일의 벡터 데이터가 생성됩니다.
  * 폴더의 DirectCloud AI 설정을 ‘사용하지 않음’으로 변경하면, 폴더에 저장된 파일의 벡터 데이터는 삭제됩니다.
  * DirectCloud AI 설정을 ‘사용하지 않음’으로 변경했다가 다시 ‘사용함’으로 되돌리면, 벡터 데이터도 다시 생성됩니다.
  * 벡터 데이터가 처리 중인 폴더에서는 DirectCloud AI 설정을 ‘사용하지 않음’으로 변경할 수 없습니다.
* 폴더 속성의 DirectCloud AI 설정은 하위 폴더에 상속되지 않습니다.\
  따라서 다음에 해당하는 폴더에 저장된 파일은 벡터 데이터가 생성되지 않습니다.
  * DirectCloud AI가 활성화된 폴더로 복사 또는 이동된 폴더
  * DirectCloud AI가 활성화된 폴더 내에서 새로 생성된 폴더
* 벡터 데이터가 생성되어 ‘AI’ 라벨이 부착된 파일을 아래 위치로 이동한 경우, 일일 배치 처리에 의해 오전 2시에 벡터 데이터가 삭제됩니다. 단, 오전 2시 이전에 원래 폴더로 되돌리면 벡터 데이터가 다시 재사용됩니다.
  * Shared Box 메뉴 내의 DirectCloud AI 설정이 비활성화된 폴더
  * Shared Box 메뉴 이외의 폴더
* DirectCloud AI가 활성화된 폴더에서 비활성화된 폴더로 파일을 복사할 경우, 벡터 데이터는 복사되지 않습니다.
* DirectCloud AI 설정이 활성화된 폴더를 이동한 경우, 벡터 데이터는 다음과 같이 처리됩니다.
  * Shared Box 메뉴 내의 비활성화된 폴더로 이동: 폴더 속성 설정이 유지되므로 폴더 내에서 생성된 벡터 데이터도 유지됩니다.
  * Shared Box 메뉴 외의 폴더로 이동: DirectCloud AI 설정 정보와 함께 벡터 데이터도 삭제됩니다.
* DirectCloud AI 설정이 활성화된 폴더를 이동한 경우, 벡터 데이터는 다음과 같이 처리됩니다.
  * Shared Box메뉴 내의 DirectCloud AI설정이 비활성화된 폴더로 이동:\
    폴더 속성의 설정은 유지되므로, 폴더 내에서 생성된 벡터 데이터는 유지됩니다.
  * Shared Box메뉴 이외의 폴더로 이동:\
    DirectCloud AI의 설정 정보가 삭제되며, 벡터 데이터도 함께 삭제됩니다.\
    폴더를 다시 Shared Box메뉴로 이동하면, DirectCloud AI설정은 기본값인 「사용하지 않음」으로 돌아갑니다.\
    삭제된 벡터 데이터는 복원할 수 없으므로 주의가 필요합니다.
* 벡터 데이터가 생성되어 AI라벨이 부여된 파일이 사용자 휴지통 또는 관리자 휴지통으로 이동될 경우, 벡터 데이터는 아래와 같이 처리됩니다.
  * 사용자 휴지통으로 이동:\
    벡터 데이터는 삭제되지 않습니다.\
    사용자 휴지통에서 DirectCloud AI가 활성화된 폴더로 파일을 복원하면, 벡터 데이터가 다시 사용됩니다.
  * 관리자 휴지통으로 이동(폴더 속성 설정에 의해 자동으로 삭제된 파일 포함):\
    일일 배치 처리에 따라 오전 2시에 삭제됩니다.\
    오전 2시 이전에 「사용자 휴지통」으로 되돌리면, 벡터 데이터는 다시 사용됩니다.
* 배치 처리로 삭제되기 전까지 벡터 데이터는 유지되지만, 파일의 「AI」라벨은 표시되지 않습니다.
* 벡터 데이터의 처리가 이루어지고 있는 폴더에서는 다음의 파일 작업을 수행할 수 없습니다.
  * 복사
  * 이동
  * 삭제(사용자 휴지통으로 이동)
* 접근 권한이 업로더인 경우에는 DirectCloud AI를 사용할 수 없으므로, 벡터 데이터 생성에 실패하더라도 파일 목록에는 실패 아이콘이 표시되지 않습니다.

**데이터 청크 사양**
{% endhint %}

**데이터 청크의 이용에 대하여**

* 데이터 청크에 대한 자세한 내용은 [DirectCloud AI의 데이터 청크를 편집하는 방법](https://help.directcloud.net/user_manual/directcloud-ai/directcloud-ai-4)을 참조하시기 바랍니다.
* 데이터 청크를 편집하려면, <마스터> 권한 또는 <전체권한>, <편집자>, <편집자－>의 접근 권한이 부여되어 있어야 합니다.
* 편집한 데이터 청크를 저장하더라도 원본 파일은 변경되지 않습니다.
* 데이터 청크가 추출된 파일에 대해 아래의 작업을 수행하더라도, 데이터 청크는 삭제되지 않고 유지됩니다.\
  ・DirectCloud AI가 활성화된 다른 Shared Box로 복사/이동\
  ・DirectCloud AI가 비활성화된 다른 Shared Box로 복사/이동\
  &#x20;   (**NOTE:** 복사/이동 대상 폴더의 DirectCloud AI를 활성화하면, 유지되어 있던 데이터 청크를 복원할 수     있습니다. )\
  ・사용자 휴지통으로 이동\
  ・관리자 휴지통으로 이동



{% hint style="info" %}
**채팅룸 이용에 관한 사양**
{% endhint %}

**질문과 답변에 대하여**

* 채팅에서 질문을 하는 사용자는, DirectCloud AI가 ‘사용함’으로 설정된 폴더의 <마스터>, <전체권한>, <편집자>, <편집자－>, 다운로더, <프리뷰어＋>, <프리뷰어>의 접근 권한이 부여되어 있어야 합니다.
* DirectCloud AI의 답변으로 반환되는 텍스트의 언어는, 질문에 입력한 텍스트의 언어와 동일합니다.
* Azure OpenAI Service의 사양에 따라, 질문 내용에 따라서는 DirectCloud AI로부터 기대한 정보가 반환되지 않거나, 부자연스러운 답변이 반환되는 경우가 있습니다.\
  이러한 경우에는 구체적인 질문을 하거나 간결한 질문을 하는 등, 질문의 형태를 바꾸어 여러 번 시도해 보시는 것을 권장합니다.
* 파일 업로드 등으로 인해 벡터 데이터가 처리 중일 때에는, 해당 파일에 관한 질문을 하더라도 답변이 반환되지 않을 수 있습니다.\
  잠시 기다리신 후 다시 질문해 주시기 바랍니다.
* 다른 웹 브라우저를 이용하여 동일한 계정으로 DirectCloud에 로그인하고, DirectCloud AI가 활성화된 폴더에 접근한 경우, 채팅 내용은 실시간으로 동기화되지 않습니다.\
  화면을 새로 고치면 동기화된 채팅 내용을 확인할 수 있습니다.
*   DirectCloud AI에서 제공이 종료된 GPT 모델이 선택되어 있는 경우, 채팅에서 질문을 입력하고 전송하면 화면 하단에 “제공이 종료된 GPT Model이 설정되어 있습니다. 관리자에게 문의해 주십시오.” 라는 오류 메시지가 표시됩니다.

    · 채팅을 이용 중인 사용자는 관리자에게 문의해 주시기 바랍니다.\
    · 관리자는 DirectCloud AI를 활성화하는 방법의 절차에 따라 GPT-4o, GPT-4o-mini, GPT-o3-mini, GPT-4.1, GPT-4.1-mini 중 하나를 선택해야 합니다.
* DirectCloud AI 채팅에서 질문을 입력할 때 입력 가능한 토큰 수를 초과하면, 그 이후의 문자는 입력할 수 없습니다. 이 경우, 오류 메시지는 표시되지 않습니다.

**추가 질문에 대하여**

* DirectCloud AI의 채팅에서는 하나의 질문에 대해 하나의 답변이 반환됩니다. \
  관리 페이지에서 ‘추가 질문’ 기능이 활성화되어 있는 경우, DirectCloud AI 채팅에서 질문할 때 ‘추가 질문’ 항목에 체크를 하면 이전에 질문했던 내용과 그에 대한 답변 텍스트를 참조할 수 있습니다.\
  ‘추가 질문’ 기능이 비활성화되어 있는 경우에는, 이전의 답변에 대해 이어서 질문할 수 없습니다. 새로운 질문으로 입력해야 합니다.\
  ‘추가 질문’ 기능을 활성화하는 절차에 대해서는 DirectCloud AI를 활성화하는 방법을 참조하시기 바랍니다.
* ‘추가 질문’을 사용할 경우, 참조되는 질문과 답변 텍스트에 포함된 단어나 문장의 종류 및 분량에 따라 토큰 또는 크레딧이 소모되므로 주의가 필요합니다.
* ‘추가 질문’에서는 최대 10개의 질문과 답변이 참조됩니다.
